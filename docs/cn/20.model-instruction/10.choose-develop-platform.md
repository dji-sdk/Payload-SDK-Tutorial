
本文旨在说明如何选择适合的飞行平台，以及在此基础上如何选择负载设备的操作系统和开发平台。选择时需要综合考虑不同平台和系统对 PSDK 功能的支持差异、负载程序的资源占用情况，以及 PSDK 支持的工具链等因素。

## 选择飞行平台

### 机型功能差异

> **说明：**<br/>
>- Matrice 300 RTK 云台口指：上置云台口（1个），下置云台口（2个）。图示可参考[设备连接](https://developer.dji.com/doc/payload-sdk-tutorial/cn/payload-quick-start/device-connect.html#m300-rtk-%E8%AE%BE%E5%A4%87%E8%BF%9E%E6%8E%A5)。
>- Matrice 400、Matrice 4E、Matrice 4TD系列飞机使用第三方开发平台仅支持基础功能，高级功能需要使用妙算3。

### 资源占用
#### Linux
使用Manifold 2-C 运行Linux 平台PSDK 示例程序，程序运行时的资源占用情况如下所示：
* 栈：约 12288 字节
* 堆：约 40960 字节
* Text 段：755359 字节
* Data 段：3872 字节
* Bss 段：23848 字节
* CPU 占用：7.2 %

#### RTOS
使用STM32F407IGH6-EVAL 运行RTOS 平台PSDK 示例程序，程序运行时的资源占用情况如下所示：
* Text 段：353620 字节
* Data 段：1836 字节
* Bss 段：85620 字节
* CPU 占用：30 %

## 选择开发平台
### 使用妙算3开发平台
妙算 3 已集成了 PSDK 开发所需的软件环境，用户可以直接在妙算 3 上进行应用开发，也可以在 PC 上搭建交叉编译环境。为此，用户可下载 [NVDIA 交叉编译工具链](https://developer.nvidia.com/embedded/jetson-linux-archive)，或自行安装 gcc-aarch64-linux-gnu 和 g++-aarch64-linux-gnu 交叉编译工具链。
