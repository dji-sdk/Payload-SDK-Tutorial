## 概述

为帮助开发者在没有 GPS 或 RTK 卫星的应用场景中实现无人机的自动化飞行控制，实现如精准悬停和避障等功能，DJI 开放了无人机视觉感知系统，包括视觉系统和六向红外感知系统以及 Matrice 400 无人机上搭载的激光雷达感知系统和六向毫米波雷达感知系统。开发者通过使用 PSDK 提供的 API 即可获取 DJI 无人机视觉感知系统的灰度图码流、激光雷达点云、毫米波雷达点云数据，结合不同的感知和定位算法，开发者能够开发出满足特定应用场景需求的高级应用程序。本篇主要用于指导开发者通过 PSDK API 获得视觉系统输出的感知灰度图、激光雷达点云以及毫米波雷达点云数据。

:::tip 说明：
* 仅通过 Linux 和 RTOS 系统开发的应用程序支持开发者使用 DJI 的感知数据获取功能。
* 使用 PSDK 的感知数据获取功能时，需使用 USB 线或网线接收无人机视觉感知系统的原始数据。
* 开发者可在[大疆官网下载中心](https://www.dji.com/cn/downloads?site=brandsite&from=nav)获得对应机型的用户手册，根据其中的部件名称介绍查看视觉感知系统的组成以及相机位置。
* Matrice 400、Matrice 4、Matrice 4D 及未来机型，使用该功能需搭配妙算3。
* 要获取无人机感知数据，需将无人机解锁起飞，或连接 DJI Assistant 2 在模拟器中将无人机解锁起飞。
:::

## 基础概念

### 无人机感知系统

DJI 无人机的视觉感知系统主要由无人机视觉感知传感器和视觉感知算法构成。在无人机飞行的过程中，感知传感器能够获取周围环境的状态，协助无人机刹车、避障和精确悬停。

- Mavic 3E/3T、Matrice 30/30T 和 Matrice 300 RTK/Matrice 350 RTK 、Matrice 4D/4T/4E、Matrice 400均支持开发者获取无人机上所有感知传感器的码流数据。
- Matrice 400支持开发者获取无人机搭载的激光雷达和毫米波雷达点云数据。

:::tip 说明：
* 在悬停期间若无人机受到外部干扰，无人机将会返回到原悬停点。
* 使用感知图像获取功能时，若无人机与无人机遥控器断开连接，无人机将会悬停。
:::

### 感知数据类型
感知图像获取功能支持开发者获取的图像类型和分辨率如下所示：

* Mavic 3E/3T：支持开发者以 20 FPS，640 × 480 的分辨率，获取上下方向 VGA 灰度图像。支持开发者以 20 FPS，480 × 480 的分辨率，获取前后左右方向 VGA 灰度图像。
* Matrice 30/30T 和 Matrice 300 RTK/Matrice 350 RTK：支持开发者以 20 FPS，640 × 480 的分辨率，获取 VGA 灰度图像。

**初始化感知模块**

```c
auto *perceptionSample = new PerceptionSample;
```

**获取感知传感器参数**

```c
returnCode = DjiPerception_GetStereoCameraParameters(&cameraParametersPacket);
```

## 使用获取感知图像的功能

使用感知图像获取功能时，开发者需要调用指定的 API 通过 USB 或 Socket 接口在应用程序运行的周期内订阅（或退订）相机视觉传感器的图像，并使用回调函数在专用的读取线程内获取订阅的图像。

### 感知图像元数据
无人机视觉传感器产生的图像主要包含帧索引和时间戳两种元数据。
* 时间戳：无人机上的视觉传感器同时共享一个计时器，有关时间同步的详细说明请参见<a href="../basic-function/time-syn.html" target="_blank" rel="noopener noreferrer">时间同步</a>。
* 帧索引：开发者订阅感知灰度图后，无人机感知灰度图将会附带帧索引。订阅或取消订阅感知灰度图，将不会改变或中断帧索引的序号。

## 使用获取感知相机参数的功能

:::tip 说明：
* 感知系统功能的使用，可以[参考代码](https://github.com/dji-sdk/Payload-SDK/tree/master/samples/sample_c%2B%2B/module_sample/perception)。
* 建议安装 OpenCV 以显示感知灰度图像。
* 为避免处理视觉传感器图像的工作阻塞主线程，请创建一个独立的线程处理无人机视觉传感器的图像。
:::



1. 创建感知码流处理线程，调用以下接口创建一个线程，用于读取感知传感器的原始码流并解码成图像。

```c
osalHandler->TaskCreate("user_perception_task", DjiTest_StereoImagesDisplayTask,
                      USER_PERCEPTION_TASK_STACK_SIZE, &s_stereoImagePacket, &s_stereoImageThread);
```

:::tip 说明：
 若开发者安装了 OpenCV 库，则可通过调用`cv::imshow()`接口显示解码后的感知灰度图像。
:::

2. 订阅感知传感器码流，例如：调用以下接口订阅前视方向的感知码流，同时在该接口中注册回调函数`DjiTest_PerceptionImageCallback`，用于处理接收到的码流。

```c
perceptionSample->SubscribeFrontImage(DjiTest_PerceptionImageCallback);
```

3. 取消订阅感知传感器码流，例如：调用以下接口取消订阅前视方向的感知码流。

```c
perceptionSample->UnSubscribeFrontImage();
```

4. 销毁线程  
调用以下接口销毁读取感知传感器码流的线程。

```c
osalHandler->TaskDestroy(s_stereoImageThread);
```
:::tip 说明：
* 由于链路带宽限制，目前所有机型只支持同时订阅两路灰度图像，且在订阅两路图像后，图像帧率会有所下降。
:::

## 使用机载激光雷达点云订阅功能
使用激光雷达点云数据获取功能时，开发者需要调用指定的 API 通过 USB 或 Socket 接口在应用程序运行的周期内订阅（或退订）相机视觉传感器的图像，并使用回调函数中接收，无人机起飞状态下可订阅到机身点云数据。

### 激光雷达点云元数据
飞行器起飞后，PSDK 可以订阅到激光雷达10hz 频率的激光雷达数据帧，每一帧激光雷达包含当前时间戳以及当前时刻下的无人机姿态信息以及多个激光雷达数据包，每个激光雷达数据包由数据包头和多个点信息组成。数据包头描述了该数据包的时间信息和点信息。

:::tip 说明：
* 参考感知相机数据订阅，为避免处理点云数据的工作阻塞主线程，请创建一个独立的线程处理无人机激光雷达点云数据。
* 激光雷达到机身 imu 外参如下：<br>
                                      [ 0.0124277 , -0.7083339 , -0.70558153,  0.01622773]    (w x y z 四元数) <br>
                                    [0.12922   ,        0    , -0.234352]               (x y z 平移量)
:::


1. 订阅机载激光雷达点云数据，例如：调用以下接口订阅机载激光雷达的点云数据，同时在该接口中注册回调函数`DjiTest_PerceptionLidarCallback`，用于处理接收到的点云信息。
```c
DjiPerception_SubscribeLidarData(DjiTest_PerceptionLidarCallback);
```

1. 取消订阅机载激光雷达点云数据，例如：调用以下接口取消订阅激光雷达点云。
```c
DjiPerception_UnsubscribeLidarData();
```

## 使用机载毫米波雷达点云订阅功能
飞行器起飞后，PSDK 可以订阅到激光雷达20hz 频率的毫米波雷达点云信息。使用毫米波雷达点云数据获取功能时，开发者需要调用指定的 API 通过 USB 或 Socket 接口在应用程序运行的周期内订阅（或退订）相机视觉传感器的图像，并使用回调函数中接收。

:::tip 说明：
参考感知相机数据订阅，为避免处理点云数据的工作阻塞主线程，请创建一个独立的线程处理无人机毫米波雷达点云数据。
:::

1. 订阅机载毫米波雷达点云数据，例如：调用以下接口订阅前向机载毫米波雷达的点云数据，同时在该接口中注册回调函数`DjiTest_PerceptionRadarCallback`，用于处理接收到的点云信息。
```c
DjiPerception_SubscribeRadarData(RADAR_POSITION_FRONT, DjiTest_PerceptionRadarCallback);
```

1. 取消订阅机载毫米波雷达点云数据，例如：调用以下接口取消订阅前向机载毫米波雷达点云。
```c
DjiPerception_UnsubscribeRadarData(RADAR_POSITION_FRONT);
```
