## 应用打包工具说明
build_dpk.sh 是一个应用打包工具，用来支持生成 xxx.dpk 应用安装包，在打包 DPK 应用时，必须要在应用的编译配置文件中调用 build_dpk.sh，并且必须要等到应用程序代码编译完生成可执行文件之后再调用这个文件以此来生成 DPK 应用包，否则可能会无法生成 DPK 安装包或者即使成功生成了 DPK 应用包也无法正常安装或运行。

build_dpk.sh 文件位于 PSDK 软件包 `tools/build_dpk/build_dpk.sh`，使用说明详见 `tools/build_dpk/README.md` 文件，具体用法为：

```bash
# Usage
    build_dpk.sh -i input app.json file
        [-o output dpk file path]
        [-h help]
    Options:
       -h        Show this help message and exit
       -i        Specify input app.json file
       -o        Specify output .dpk file path
```

## 应用编译和打包步骤
### 1. 修改应用代码
首先需要通过开发者用户中心，获得所创建 PSDK-妙算3应用的详情信息，将这些信息填入 PSDK 开发包的指定文件中：

* samples/sample_c/platform/linux/manifold3/application/dji_sdk_app_info.h
* samples/sample_c++/platform/linux/manifold3/application/dji_sdk_app_info.h
```c
#define USER_APP_NAME               "your_app_name"
#define USER_APP_ID                 "your_app_id"
#define USER_APP_KEY                "your_app_key"
#define USER_APP_LICENSE            "your_app_license"
#define USER_DEVELOPER_ACCOUNT      "your_developer_account"
#define USER_BAUD_RATE              "460800"
```

接着还需要在 PSDK 软件开发包的指定文件中填写应用的 firmwareVersion 版本信息：

```c
{

```

* samples/sample_c++/platform/linux/manifold3/application/application.cpp
```c++
void Application::DjiUser_ApplicationStart()
```

之后就可以按照应用功能需求来添加修改其他的文件了（如源码文件、配置文件等），从而实现相关应用功能。

### 2. 创建 app.json 应用级配置文件
参考`samples/sample_c/platform/linux/manifold3/app_json/app.json`文件，创建一个自己的 app.json 应用级配置文件，配置文件填写规范详见`samples/sample_c/platform/linux/manifold3/app_json/README.md`文件或者上述 <a href = "#app-json-config">app.json 应用级配置文件说明</a> 章节内容。

### 3. 修改 CMakeLists.txt 编译配置文件

参考示例应用程序中的`samples/sample_c/platform/linux/manifold3/CMakeLists.txt`编译构建文件，直接在这个文件中调用应用打包脚本 build_dpk.sh，输入 app.json 应用级配置文件的路径，以及 xxx.dpk 应用安装包的生成路径，编译后便可在应用安装包的生成路径中生成 xxx.dpk 应用安装包。

在应用开发时，您可以根据您的需要和具体的代码结构在 CMakeLists.txt 编译构建文件中修改 app.json 应用配置文件的路径`JSON_FILE`和 xxx.dpk 应用安装包的生成路径`OUTPUT_PACKAGE`，最后打包生成的 xxx.dpk 应用安装包就会位于你所填写的`OUTPUT_PACKAGE`路径下。
```cmake
# set app.json and build_dpk.sh path
set(JSON_FILE ${CMAKE_CURRENT_SOURCE_DIR}/app_json/app.json)
set(SCRIPT_FILE ${CMAKE_CURRENT_SOURCE_DIR}/../../../../../tools/build_dpk/build_dpk.sh)

# set output dpk path
set(OUTPUT_PACKAGE ${CMAKE_BINARY_DIR}/dpk)

# generate .dpk app package
add_custom_command(TARGET ${PROJECT_NAME}
    POST_BUILD # generate .dpk package after make
    COMMAND bash ${SCRIPT_FILE} -i ${JSON_FILE} -o ${OUTPUT_PACKAGE}
    DEPENDS ${JSON_FILE}
    COMMENT "Running build_dpk.sh, generating .dpk app package from app.json file"
)
```

### 4. 打包生成 xxx.dpk 应用安装包
PSDK 软件包支持 CMake 编译，支持编译后直接生成 xxx.dpk 应用安装包，您可以通过 cmake 命令来打包生成 xxx.dpk 应用安装包，命令如下：
```bash
cd Payload-SDK
mkdir build
cd build
cmake ..
make
```

命令执行成功后，在`OUTPUT_PACKAGE`路径下便会生成 xxx.dpk 应用安装包，即妙算应用安装包，得到它之后就可以安装和运行相应的妙算应用了。

## 应用安装测试
获得 DPK 应用安装包之后，安装/更新 DPK 应用到妙算设备内，安装成功之后，将会在妙算3设备内 /open_app 分区生成一个目录，这个目录就是应用目录，应用的文件（包括可执行文件、配置文件等）都在这个目录内，当应用更新成功之后，这个目录下的文件会随之更新。

如果想要测试安装的这个 DPK 应用，则需要启停这个 DPK 应用，然后观察它的运行状态是否正常，并且运行场景是否符合预期，这可以在 PILOT APP 上进行，如果发现运行状态不对或者运行场景不符合预期，可以通过应用日志来对应用进行调试。

## FAQ 问题汇总

**1. 配置文件的路径在 app.json 文件中怎么填写？妙算应用运行时如何使用打包后的配置文件？**

应用打包和应用运行是解耦的。

应用打包是将应用相关的配置文件、可执行文件等按照一定的文件结构组织起来，形成一个 xxx.dpk 应用安装包，之所以要将配置文件的路径写到 app.json 文件，是为了在打包时将配置文件 copy 到 xxx.dpk 应用安装包中，app.json 文件中 userconfig 字段的值就是配置文件的路径，具体值为代码中配置文件相对于 app.json 文件的相对路径，可以写整个目录，也可以写具体的文件。

对于应用运行，当安装 xxx.dpk 应用程序包成功后，在负载设备（如妙算3） /open_app 分区会生成一个应用目录，应用相关的文件会保存在这个应用目录中，当应用运行时读取或者写这个应用目录中的文件，读写的具体路径和 app.json 文件中填写的 userconfig 字段的值相关，如果写的是整个目录，则需要读写目录下各个文件，而如果写的是具体的文件，则只需要读写具体的这个文件就行。
假如应用程序要读写的配置文件在代码中的路径为 `/cfg/1/1.cfg`, `/cfg/2/2.cfg` , `/3/3.cfg`，app.json 文件在代码中的路径为 `/json/app.json`，有效的 `userconfig` 格式和应用运行时路径包括：

**2. 安装妙算 PSDK 应用安装包 xxx.dpk 文件失败，报错：启动应用服务失败，请重启设备后重试。**

**解决方法：**
- 检查应用代码是否正确，所使用的第三方依赖库是否为静态库而非动态库，妙算应用在安装的时候需要启动一下，如果应用程序代码不对或者使用了动态库的话，安装的时候将无法正常启动，必须要保证 xxx.dpk 应用安装包中的程序能够正常启动。
- 应用安装时的应用启动日志保存在负载设备内（如妙算3）/blackbox/system/app_temp_files 目录下，可基于此排查应用是否正常启动以及如果没有正常启动的话无法正常启动的原因。
- 如果发现 xxx.dpk 应用安装包中的程序无法正常启动的话，需要修改代码，之后重新编译打包，重新安装；而如果发现程序可以正常启动无报错，但仍然有安装应用失败启动应用服务失败的报错的话，可以重启设备后重试或者联系 DJI 进行排查。

**3. 安装妙算 PSDK 应用安装包 xxx.dpk 文件失败，报错：校验应用信息失败，请确保应用信息符合规范。**

**解决方法:**
- 首先检查 app.json 文件中填写的 user_app_id 和 firmware_version 字段值是否符合填写规范，参考`app.json 应用级配置文件`，即 user_app_id 的值要和代码中的 USER_APP_ID 保持一致并且 firmware_version 的值要和代码中的 firmwareVersion 保持一致，如果任一字段值不一致，需要修改为一致，之后重新打包，重新安装。
- 如果检查上述字段值都一致的话，需要再检查应用安装时是否有正常启动无报错，应用安装时的启动日志详见负载设备（如妙算3）内 /blackbox/system/app_temp_files 目录下的文件。

**4. 应用安装成功后，启动应用失败或者虽然启动应用成功但是应用运行现象与预期不符：**

**解决方法：**
- 请确认应用的依赖文件都打包到了 xxx.dpk 文件中，并且代码中依赖文件的读写路径正确，依赖文件的正确读写路径参考：`问题1.配置文件的路径在 app.json 文件中怎么填写？妙算 PSDK 应用运行时如何使用打包后的配置文件？`中的解决方法。
- 如果应用的依赖文件读写路径正确，并且所有依赖文件都打包到了 xxx.dpk 文件中，则可以查看应用的运行日志来 debug 应用启动失败或者启动成功但现象与预期不符的原因。

**5. 应用依赖三方库，安装三方库时提示权限问题**

**解决方法：**
- 不支持 root 账户，安装依赖库请以静态库或源码方式包含进三方应用中。
