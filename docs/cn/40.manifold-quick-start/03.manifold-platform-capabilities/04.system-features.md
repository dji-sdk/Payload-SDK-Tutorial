## 固件升级功能
支持通过 DJI 调参软件和 DJI PILOT APP 升级妙算3固件，推荐使用调参软件升级妙算3固件版本。

## 访问机载网络
该功能允许妙算3应用通过飞机的机载4G 网络访问互联网，需要飞机安装 DJI 增强图传模块并开通相应流量包。
默认情况下，负载网络访问功能已经开启，无需进行任何操作（依赖增强图传模块安装在飞机上，且4G 网络功能正常）。 支持 Matrice 400、Matrice 4TD/4D、Matrice 4T/4E 三款飞机。
更多详细功能描述可以参考 <a href="../../function-overview/advanced-function/access-internet.html" target="_blank" rel="noopener noreferrer">使用机载网络</a>。
## 高功率输出
妙算3尾部 TypeC 接口可通过 PSDK API 实现高压输出（高压输出时请勿连接标准 USB 外设避免损毁，如 U 盘）  
PSDK API 接口：DjiPowerManagement_OutputHighPower   
- 连接 Matrice 400：14V / 5A
- 连接 Matrice 4E/4T：12.8V-17V / 3.38A
- 连接 Matrice 4D/4TD：16.8V-25.5V / 3A 
## 功率自适应
妙算 3 采用分级功耗管理策略，根据系统运行状态在功耗与性能之间自动平衡，以满足行业无人机长时间作业需求。
- 待机状态：当无应用业务运行或系统负载较低时，系统采用低功耗策略，限制 CPU 使用至 4 个 CPU 核心，并根据负载自适应调整频率，在保证基础功能稳定运行的同时最大限度降低功耗。 
- 业务运行状态：当应用业务启动或负载升高时，系统自动切换至 MAXN（最大性能）模式，启用全部 8 个 CPU 核心， 并同时释放 CPU、GPU 与 DDR 的完整性能能力，并由 NVIDIA 平台根据实时负载动态调度各模块频率，确保业务稳定运行。 

功耗模式切换由系统自动完成，开发者无需干预，可专注于应用功能和业务功能开发。  

#### 妙算 3 功率自适应管理策略测试方式与示例 
启动和停止 DPK 应用需要通过 Pilot 或者司空界面操作功耗策略才能生效。

| 运行状态 | 测试方式 | 示例场景 | 预期现象 |
| :-- | :-- | :-- | :-- |
| 待机状态 | 不启动任何 DPK 应用，或应用已安装但未运行 | 设备上电后处于空闲状态 | CPU 使用限制在 4 核心，频率随负载自适应调整，整体功耗较低。 |
| 业务运行状态（MAXN） | 启动 DPK 应用并触发实际业务负载 | 启动 AI 识别、图像处理等应用 | 系统自动切换至 MAXN 模式，8 个 CPU 核心全部启用，CPU / GPU / DDR 性能释放，频率随负载动态调度。 |
| 状态切换验证 | 启停 DPK 应用，观察系统状态变化 | 启动应用 → 执行业务 → 停止应用 | 系统在 4 核低功耗模式 ↔ 8 核 MAXN 模式间自动切换。 |

**测试示例**  
- 在 Pilot APP 中手动关闭 DPK 应用，在 CLI 控制台中输入 nproc 命令，可以看到显示4核。
- 在 Pilot APP 中手动开启 DPK 应用，在 CLI 控制台中输入 nproc 命令，可以看到显示8核。
