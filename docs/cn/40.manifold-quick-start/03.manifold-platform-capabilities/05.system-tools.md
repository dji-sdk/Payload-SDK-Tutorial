## 网络配置工具
妙算 3 扩展接口支持外接 USB 以太网卡、USB Wi-Fi 网卡以及 4G/5G CPE 设备，以满足多样化的网络接入需求。通过 netctl.sh 脚本工具，可对网络接口进行统一管理，包括配置固定 IP 地址、连接 Wi-Fi 网络以及设置多网卡的路由优先级。

### 基础说明

`netctl.sh` 是一个基于 NetworkManager（`nmcli`）的网络配置工具脚本，用于在妙算 3 上进行设备级网络配置，主要支持：

- WiFi 连接与断开（Open / WPA2 / WPA3-PSK）
- WiFi / 有线网络的静态 IP 与 DHCP 配置
- 基于 NetworkManager 的声明式路由与多出口优先级配置（支持链路失效自动切换）

:::warning 注意：
本工具不处理任何需要人工交互的认证流程，只负责网络设备层面的配置。
:::

#### 基本用法

```bash
netctl.sh <command> [options]
```

#### 命令概览

- 扫描 WiFi

```bash
netctl.sh scan_wifi [--iface IFACE]
```

- 连接 WiFi

```bash
netctl.sh connect_wifi --ssid <SSID> [--password <PSK>] [--iface IFACE]
```

- 断开 WiFi 连接

```bash
netctl.sh disconnect_wifi --ssid <SSID> [--iface IFACE]
```

- 配置静态 IP

```bash
netctl.sh setup_static_ip --type <TYPE> [--ssid <SSID>] [--iface IFACE] --ip <IP> --gw <Gateway> [--mask <CIDR>] [--dns DNS1,DNS2]
```

- 恢复 DHCP 模式

```bash
netctl.sh setup_dhcp_ip --type <TYPE> [--ssid <SSID>] [--iface IFACE]
```

- 配置路由

```bash
netctl.sh setup_route --route "NET>TYPE" [--route "..."] [--iface TYPE:IFACE]
```

- 恢复路由配置

```bash
netctl.sh setup_route --reset
```

- 查看简要帮助

```bash
netctl.sh -h
netctl.sh --help
```

- 查看完整帮助说明

```bash
netctl.sh --help-full
```

#### `--iface` 说明（多网卡与接口选择策略）

:::tip 说明：
 建议在使用该参数指定接口之前，先通过 `ifconfig` 命令确认各网络接口的实际命名，以避免因接口名称填写错误导致 `netctl.sh` 执行失败。
:::
- 当指定的 `TYPE` 仅对应一个可用接口时，系统将自动选择该接口，无需指定 `--iface` 参数。
- 当指定的 `TYPE` 对应多个可用接口时，必须通过 `--iface` 参数显式指定所使用的接口；若未指定 `--iface` 且存在多个接口，命令将直接报错并退出。

### 认证与安全边界说明

本工具是设备级网络配置工具，而非用户会话管理工具。连接 WiFi 时：

✅ 支持：
- Open WiFi
- WPA2 / WPA3-PSK
- 普通以太网直连

❌ 不支持：
- Captive Portal / Web 登录
- 短信 / 动态密码
- 802.1X（EAP / PEAP / TLS / TTLS）

是否需要在断线或重启后重新认证，完全由网络自身策略决定，`netctl.sh` 不参与控制。

### WiFi 管理

#### 扫描 WiFi

```bash
netctl.sh scan_wifi [--iface IFACE]
```

##### 参数说明

| 参数 | 是否必须 | 说明 | 示例 |
| :-- | :-- | :-- | :-- |
| `--iface` | 否 | 指定接口。<ul><li> 当 `TYPE` 仅有一个接口时自动使用；</li><li>当存在多个接口时，需通过 `--iface` 指定接口。</li></ul>**注意：** 建议在使用该参数指定接口之前，先通过 `ifconfig` 命令确认各网络接口的实际命名，以避免因接口名称填写错误导致 `netctl.sh` 执行失败。 | `--iface wlan0` |



#### 连接 WiFi

```bash
netctl.sh connect_wifi --ssid <SSID> [--password <PSK>] [--iface IFACE]
```

##### 参数说明

| 参数 | 是否必须 | 说明 | 示例 |
| :-- | :-- | :-- | :-- |
| `--ssid` | 是 | 要连接的 WiFi SSID 名称 | `--ssid Airport_WiFi` |
| `--password` | 否 | WiFi 密码。不指定时脚本会交互式隐藏输入。指定时用于自动化脚本（请注意明文安全风险） | `--password 'MyPass123!'` |
| `--iface` | 否 | 指定接口。<ul><li> 当 `TYPE` 仅有一个接口时自动使用；</li><li>当存在多个接口时，需通过 `--iface` 指定接口。</li></ul>**注意：** 建议在使用该参数指定接口之前，先通过 `ifconfig` 命令确认各网络接口的实际命名，以避免因接口名称填写错误导致 `netctl.sh` 执行失败。 | `--iface wlan0` |

##### 行为说明（重要）

- 未提供 `--password` 时：
  - 若检测为开放 WiFi，直接连接。
  - 若为加密 WiFi，会安全提示输入密码。

- 成功连接后将：
  - 创建或复用 NetworkManager 连接配置。
  - 启用自动重连（包括重启后）。

##### 上网优先级说明（重要）

`connect_wifi` 命令只负责建立 WiFi 连接，不会自动修改系统的路由或上网优先级。  
如果系统同时连接了：Onboard（机身网络）+ Ethernet（有线网络）+ WiFi（无线网络），默认情况下以太网和机身网络的路由优先级高于 WiFi（详见<a href="#上网优先级与出口选择说明">上网优先级与出口选择说明</a>）。

此时即使 WiFi 已成功连接，系统的互联网流量仍然可能继续走机身网络或者以太网。如果希望通过 WiFi 上网，请在 WiFi 连接成功后，执行 `netctl.sh setup_route` 子命令显式配置路由优先级。



#### 断开 WiFi

```bash
netctl.sh disconnect_wifi --ssid <SSID> [--iface IFACE]
```

##### 参数说明

| 参数 | 是否必须 | 说明 | 示例 |
| :-- | :-- | :-- | :-- |
| `--ssid` | 是 | 要断开连接的 WiFi SSID 名称 | `--ssid Airport_WiFi` |
| `--iface` | 否 | 指定接口。<ul><li> 当 `TYPE` 仅有一个接口时自动使用；</li><li>当存在多个接口时，需通过 `--iface` 指定接口。</li></ul>**注意：** 建议在使用该参数指定接口之前，先通过 `ifconfig` 命令确认各网络接口的实际命名，以避免因接口名称填写错误导致 `netctl.sh` 执行失败。 | `--iface wlan0` |

##### 行为说明

- 断开指定 SSID 的 WiFi 连接。
- 同时禁用该连接的自动重连。
- 不删除 NetworkManager 中的连接配置。

##### 执行结果校验

在执行 `netctl.sh disconnect_wifi` 命令后，建议对当前网络配置进行检查，以确认断开操作已生效，且设备网络状态符合预期。

**1. 检查当前网络配置状态**

可通过以下命令查看当前生效的网络接口状态及配置信息：

```bash
nmcli device show
```

重点关注：

- WiFi 接口是否已处于 `disconnected` 状态
- 当前是否仍存在有效的 IP 地址或默认路由

**2. 验证网络连通性**

在确认 WiFi 已断开后，可通过 `ping` 命令验证设备是否仍具备网络连通能力（例如是否切换至有线网络或其他接口）：

```bash
ping -c 3 <目标地址>
```

若 `ping` 无响应，说明设备已成功断开外网连接；若仍有响应，则表示当前存在其他可用网络接口。

### IP 配置

#### 配置静态 IP

```bash
netctl.sh setup_static_ip --type <TYPE> [参数...]
```

- WiFi

```bash
netctl.sh setup_static_ip --type wifi [--iface IFACE] --ssid <SSID> --ip <IP> --gw <Gateway> [--mask <CIDR>] [--dns DNS1,DNS2]
```

- Ethernet / Cellular

```bash
netctl.sh setup_static_ip --type eth|cellular [--iface IFACE] --ssid <SSID> --ip <IP> --gw <Gateway> [--mask <CIDR>] [--dns DNS1,DNS2]
```

**参数说明**

| 参数 | 是否必须 | 说明 | 示例 |
| :-- | :-- | :-- | :-- |
| `--type` | 是 | 接口类型：`wifi` / `eth` / `cellular` | `--type wifi` |
| `--ssid` | WiFi 场景必须（`--type wifi`） | WiFi SSID 名称，用于为该 SSID 的 connection 配置 IP（仅 WiFi 场景需要，有线场景不需要） | `--ssid Airport_WiFi` |
| `--iface` | 否 | 指定接口。<ul><li> 当 `TYPE` 仅有一个接口时自动使用；</li><li>当存在多个接口时，需通过 `--iface` 指定接口。</li></ul>**注意：** 建议在使用该参数指定接口之前，先通过 `ifconfig` 命令确认各网络接口的实际命名，以避免因接口名称填写错误导致 `netctl.sh` 执行失败。 | `--iface wlan0`<br>`--iface eth0` |
| `--ip` | 静态 IP 场景必填 | 静态 IP 地址。如果省略，表示不改 IP（继续使用现有的 DHCP 或静态 IP） | `--ip 192.168.1.100` |
| `--mask` | 静态 IP 场景推荐 | 子网掩码前缀长度 | `--mask 24` |
| `--gw` | 静态 IP 场景必填 | 默认网关（路由 IP），必须配合 `--ip` 使用 | `--gw 192.168.1.1` |
| `--dns` | 否 | DNS 服务器列表，逗号分隔。若不指定，保留原有 DNS；指定则覆盖 | `--dns 4.4.4.4,8.8.8.8` |



:::warning 注意事项

（重要）脚本在配置路由和联网时不会自动纠正或检测网关合法性，请用户自行保证配置正确，配置静态 IP 时一定要确保符合以下规则，否则配置 IP 之后可能会导致无法正常上网。
:::

- **IP 地址（`--ip`）**
  - 必须位于当前局域网网段内。
  - 不能与网络中其他设备的 IP 冲突。
  - 不能使用网络地址或广播地址。

- **子网掩码（`--mask`）**
  - 使用 CIDR 表示法。
  - 常见取值：
    - `24` → `255.255.255.0`（最常见）
    - `16` → `255.255.0.0`
    - `8` → `255.0.0.0`

- **默认网关（`--gw`，非常重要）**
  - 网关必须：
    - 与 `--ip` 位于同一子网。
    - 且真实存在、可达。
  - 通常是：
    - 家庭/办公网络：路由器 IP（如 `192.168.1.1`）
    - 工业网络：上级交换机或核心路由 IP

**行为说明**

- 仅修改 NetworkManager 中的 `ipv4` 配置。
- 若连接当前处于 `active` 状态，脚本会先断开网络，之后重连使配置立即生效。
- 若未激活，则在下次连接时生效。

**参数说明**

### 路由配置（声明式，支持自动切换）

#### 设置路由规则

```bash
netctl.sh setup_route --route "NET>TYPE" [--route "..."] [--iface TYPE:IFACE]
```

**参数说明**

| 参数 | 是否必须 | 说明 | 示例 |
| :-- | :-- | :-- | :-- |
| `--route "NET>TYPE"` | 是 | 路由规则，可多次指定。<ul><li>`NET` 格式：`default` → `0.0.0.0/0`，或 CIDR 网段（如 `192.168.1.0/24`）</li><li>`TYPE`：`eth` \| `wifi` \| `cellular` \| `onboard`</li></ul> | `--route "default>onboard"`<br>`--route "192.168.42.0/24>wifi"`<br>`--route "4.4.4.4>cellular"` |
| `--iface TYPE:IFACE` | 否 | 绑定指定接口给某个 `TYPE`。规则：<ul><li>当 `TYPE` 仅有一个接口时自动使用；</li><li>当存在多个接口时，需通过 `--iface TYPE:IFACE` 指定接口；</li><li> 可多次指定，用于不同 `TYPE`；</li><li>同一 `TYPE` 重复指定时，最后一次生效并给出警告。</li></ul>**注意：** 建议在使用该参数指定接口之前，先通过 `ifconfig` 命令确认各网络接口的实际命名，以避免因接口名称填写错误导致 `netctl.sh` 执行失败。 | `--iface wifi:wlan0`<br>`--iface eth:eth1`<br>`--iface onboard:l4tbr0` |

---

##### onboard 路由的重要限制

在妙算 3 系统中：

- `onboard` 是黑盒出口
- NAT / 转发由无人机完成
- 设备侧没有可见的三层网关

由此产生的强制规则：

✅ 支持：

```bash
netctl.sh setup_route --route "default>onboard"
```

❌ 不支持（会被跳过）：

```bash
netctl.sh setup_route --route "1.1.1.1>onboard"
netctl.sh setup_route --route "192.168.140.0/24>onboard"
```

这是 Linux 路由语义限制，属于**设计行为而非缺陷**。

**示例**

- 所有流量通过 onboard：

```bash
netctl.sh setup_route --route "default>onboard"
```

- 10.0.0.0/8 目标网络走有线，4.4.4.4 目标网络走蜂窝网络，192.168.42.0/24 目标网络走 WiFi，其余目标网络走 onboard 机身网络：

```bash
netctl.sh setup_route --route "10.0.0.0/8>eth" --route "4.4.4.4>cellular" --route "192.168.42.0/24>wifi" --route "default>onboard"
```

- 多 WiFi 网卡，指定 `wlan1` 作为出口：

```bash
netctl.sh setup_route --iface wifi:wlan1 --route "default>wifi"
```

- 多出口系统，分别绑定接口：

```bash
netctl.sh setup_route --iface eth:eth1 --iface wifi:wlan0 --route "default>eth" --route "default>wifi"
```

**行为说明**

- 基于 NetworkManager 声明式路由。
- 同类型接口使用相同 `metric`。
- 当出口接口断开，其路由自动失效，其他出口自动接管。

---

#### 重置路由

```bash
netctl.sh setup_route --reset
```

**行为说明**

- 仅移除 `netctl.sh` 添加的路由。
- 恢复 NetworkManager 的默认 `metric` 配置。
- 不影响非 `netctl` 管理的系统路由。


### 上网优先级与出口选择说明

当同一个目标网络存在多个 `TYPE` 路由出口时，系统会根据路由 `metric` 自动决定上网出口：

- `metric` 越小，优先级越高。
- 当高优先级接口断开时，系统会自动切换到下一个可用出口。
- 无需手动切换或重新配置。
- `netctl` 配置的优先级顺序为：`cellular > eth > onboard > wifi`

#### 示例

**主出口 + 自动回退：**

```bash
netctl.sh setup_route --route "default>eth" --route "default>onboard" --route "default>wifi"
```

**行为：**

- Ethernet 可用 → 使用 Ethernet
- Ethernet 断开 → 自动切换到 onboard
- onboard 不可用 → 自动切换到 WiFi

### 典型使用流程建议

- 使用 `connect_wifi` 建立基础联网。
- 配置静态 IP 或 DHCP（如有需要）。
- 使用 `setup_route` 配置出口；如出现问题，可使用 `setup_route --reset` 恢复系统默认行为。

## 依赖安装工具
为了帮助开发者在无 root 环境下快速、低门槛地集成并调用主应用所需的第三方依赖，我们在妙算 3 内置了安装脚本 `install.py`，可快速完成依赖的下载与安装。

### 脚本介绍
本脚本用于非 root 权限下安装软件（rootless），提供 apt 和 deb 两种安装方式。
- apt：基于 APT 源解析并安装包。
- deb：从 `.deb` 文件/URL 安装包。

:::tip 说明：
- 当采用 apt 子命令安装软件时，支持自定义 APT 源与 GPG 密钥。
- 支持自定义数据与数据库保存目录（默认路径为/open_app/install）。
:::


### 运行条件
- Python 3.7 及以上版本。
- 网络可用（访问 APT 源或`密钥.deb URL`）

### 使用说明
- 查看帮助：

    ```bash
    install.py -h
    ```

- 查看 apt 子命令的帮助信息：

    ```bash
    install.py apt -h
    ```

- 查看 deb 子命令的帮助信息：

    ```bash
    install.py deb -h
    ```

- 具体使用命令参考如下格式：
```bash  
install.py [sub_command（apt/deb）] [params ...]
```

#### APT 安装方式

```bash
install.py apt [package ...] [--source/-s <源> ...] [--gpg-key/-k <密钥> ...] [--dir/-d <目录>]
```

参数说明：

| 参数 | 是否必选 | 说明 |
|------|----------|------|
| `package` | 必选 | 要安装的软件包名称列表，空格分隔多个，例如：`curl git vim` |
| `--source` / `-s` | 可选 | 自定义 APT 源，用于指定软件包来源（仓库路径），可重复使用以添加多个源。<br><br>APT 源为文件路径（建议使用字符串格式）：<br>`--source "/open_app/test/emqx_emqx-enterprise5.list"`<br><br>APT 源为字符串（需使用引号）：<br>`--source "deb https://mirrors.aliyun.com/ros2/ubuntu/ focal main"` |
| `--gpg-key` / `-k` | 可选 | APT 源验证密钥，用于对软件包来源进行授权。可为密钥文件路径或下载 URL，可重复使用以添加多个密钥。<br><br>文件路径：<br>`--gpg-key /open_app/emqx_emqx-enterprise5-archive-keyring.gpg`<br><br>下载 URL（需使用引号）：<br>`--gpg-key "https://raw.githubusercontent.com/ros/rosdistro/master/ros.asc"` |
| `--dir` / `-d` | 可选 | 软件安装目录。安装完成后的软件数据和数据库文件（`data.db`）将保存在该目录中。 |




#### deb 安装方式

```bash
install.py deb [--deb_source <路径或 URL> ...] [--dir/-d <目录>]
```

参数说明：

| 参数 | 是否必选 | 说明 |
|------|----------|------|
| `--deb_source` | 必选 | 要安装的软件的 `.deb` 文件路径或 URL，可重复使用以安装多个应用，例如：<br><br>本地文件路径：<br>`--deb_source /open_app/curl_7.68.0-1ubuntu2.25_amd64.deb`<br><br>URL：<br>`--deb_source https://dl.min.io/server/minio/release/linux-arm64/minio_20250907161309.0.0_amd64.deb` |
| `--dir` / `-d` | 可选 | 软件的安装目录，安装完成的软件数据和数据库文件（`data.db`）将保存在这个目录中。 |



### 运行示例

脚本运行过程中会列出需要新安装的各个软件，系统中已有或者已被记录到用户自定义数据库中的软件不会再被安装。
#### APT 安装：

- 使用系统源，指定安装目录（/open_app/user_install）安装 gdebi：

    ```bash
    install.py apt gdebi -d /open_app/user_install
    ```

- 添加 ROS2 源和密钥，默认目录安装 curl、htop 和 ros-foxy-ros-base：
  - `curl htop ros-foxy-ros-base`：需要安装的软件列表。
  - `--source "deb https://mirrors.aliyun.com/ros2/ubuntu/ focal main"`：添加 ROS2 源，APT 源为字符串。
  - `--gpg-key "https://raw.githubusercontent.com/ros/rosdistro/master/ros.asc"`：指定 APT 源的密钥，使用 URL 方式下载密钥。
  
  ```bash
  install.py apt curl htop ros-foxy-ros-base --source "deb https://mirrors.aliyun.com/ros2/ubuntu/ focal main" --gpg-key "https://raw.githubusercontent.com/ros/rosdistro/master/ros.asc"
  ```

- 添加多个源和多个密钥，默认目录安装 emqx 和 ros-foxy-ros-desktop：
  - `ros-foxy-desktop emqx-enterprise`：需要安装的软件列表。
  - `--source "/open_app/test/emqx_emqx-enterprise5.list"`：添加 emqx 源，APT 源为文件路径。
  - `--source "deb https://mirrors.aliyun.com/ros2/ubuntu/ focal main"`：添加 ROS2 源，APT 源为字符串。
  - `--gpg-key /open_app/emqx_emqx-enterprise5-archive-keyring.gpg`：指定 APT 源的密钥，使用文件路径方式获取密钥。
  - `--gpg-key "https://raw.githubusercontent.com/ros/rosdistro/master/ros.asc"`：指定 APT 源的密钥，使用 URL 方式下载密钥。

  ```bash
  install.py apt ros-foxy-desktop emqx-enterprise --source "/open_app/test/emqx_emqx-enterprise5.list" --source "deb https://mirrors.aliyun.com/ros2/ubuntu/ focal main" --gpg-key /open_app/emqx_emqx-enterprise5-archive-keyring.gpg --gpg-key "https://raw.githubusercontent.com/ros/rosdistro/master/ros.asc"
  ```

#### deb 安装
文件路径 + URL：

```bash
python3 install.py deb --deb_source "https://dl.min.io/server/minio/release/linux-arm64/minio_20250907161309.0.0_arm64.deb"  --deb_source /open_app/curl_7.68.0-1ubuntu2.25_arm64.deb --dir /open_app/user_install
```

### 常见问题与排查
1. 包找不到 `(N: Unable to locate package <pkg>)`   
确认 `--source` 参数正确，并且源中确实包含该包。
2. GPG 错误 `(NO_PUBKEY ... / is not signed)`  
   - 确认使用了正确的密钥 `(--gpg-key)`，建议 `--source` 参数中不要使用 `signed-by`（仓库行中），脚本会自行处理。
   - 如果使用的是密钥文件，确保密钥可读且路径正确。
3. 网络/证书问题 `(Unable to establish SSL connection 或 Failed to fetch)`  
使用 `curl -v <URL>` 或 `wget --spider <URL>` 验证连通性。
