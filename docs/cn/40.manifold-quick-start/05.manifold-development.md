## 快速集成第三方依赖 
为了帮助开发者在无 root 环境下快速、低门槛地集成并调用主应用所需的第三方依赖，我们提供了安装脚本 [install.py](https://terra-1-g.djicdn.com/71a7d383e71a4fb8887a310eb746b47f/psdk/install.py)，可快速完成依赖的下载与安装。

### 脚本介绍
本脚本用于非 root 权限下安装软件（rootless），提供 apt 和 deb 两种安装方式。
- apt：基于 APT 源解析并安装包。
- deb：从 `.deb` 文件/URL 安装包。

:::tip 说明：
- 当采用 apt 子命令安装软件时，支持自定义 APT 源与 GPG 密钥。
- 支持自定义数据与数据库保存目录（默认路径为/open_app/install）。
:::


### 运行条件
- Python 3.7 及以上版本。
- 网络可用（访问 APT 源或`密钥.deb URL`）

### 使用说明
- 查看帮助：

    ```bash
    python3 install.py -h
    ```

- 查看 apt 子命令的帮助信息：

    ```bash
    python3 install.py apt -h
    ```

- 查看 deb 子命令的帮助信息：

    ```bash
    python3 install.py deb -h
    ```

具体使用命令参考如下格式：  
    `python3 install.py [sub_command（apt/deb）] [params ...] `

### 运行示例

脚本运行过程中会列出需要新安装的各个软件，系统中已有或者已被记录到用户自定义数据库中的软件不会再被安装。
#### APT 安装：

- 使用系统源，指定安装目录（/open_app/user_install）安装 gdebi：

    ```bash
    python3 install.py apt gdebi -d /open_app/user_install
    ```

- 添加 ROS2 源和密钥，默认目录安装 curl、htop 和 ros-foxy-ros-base：
  - `curl htop ros-foxy-ros-base`：需要安装的软件列表。
  - `--source "deb https://mirrors.aliyun.com/ros2/ubuntu/ focal main"`：添加 ROS2 源，APT 源为字符串。
  - `--gpg-key "https://raw.githubusercontent.com/ros/rosdistro/master/ros.asc"`：指定 APT 源的密钥，使用 URL 方式下载密钥。
  
  ```bash
  python3 install.py apt curl htop ros-foxy-ros-base --source "deb https://mirrors.aliyun.com/ros2/ubuntu/ focal main" --gpg-key "https://raw.githubusercontent.com/ros/rosdistro/master/ros.asc"
  ```

- 添加多个源和多个密钥，默认目录安装 emqx 和 ros-foxy-ros-desktop：
  - `ros-foxy-desktop emqx-enterprise`：需要安装的软件列表。
  - `--source "/open_app/test/emqx_emqx-enterprise5.list"`：添加 emqx 源，APT 源为文件路径。
  - `--source "deb https://mirrors.aliyun.com/ros2/ubuntu/ focal main"`：添加 ROS2 源，APT 源为字符串。
  - `--gpg-key /open_app/emqx_emqx-enterprise5-archive-keyring.gpg`：指定 APT 源的密钥，使用文件路径方式获取密钥。
  - `--gpg-key "https://raw.githubusercontent.com/ros/rosdistro/master/ros.asc"`：指定 APT 源的密钥，使用 URL 方式下载密钥。

  ```bash
  python3 install.py apt ros-foxy-desktop emqx-enterprise --source "/open_app/test/emqx_emqx-enterprise5.list" --source "deb https://mirrors.aliyun.com/ros2/ubuntu/ focal main" --gpg-key /open_app/emqx_emqx-enterprise5-archive-keyring.gpg --gpg-key "https://raw.githubusercontent.com/ros/rosdistro/master/ros.asc"
  ```

#### deb 安装
文件路径 + URL：

```bash
python3 install.py deb --deb_source "https://dl.min.io/server/minio/release/linux-arm64/minio_20250907161309.0.0_arm64.deb"  --deb_source /open_app/curl_7.68.0-1ubuntu2.25_arm64.deb --dir /open_app/user_install
```

### 常见问题与排查
1. 包找不到 `(N: Unable to locate package <pkg>)`   
确认 `--source` 参数正确，并且源中确实包含该包。
2. GPG 错误 `(NO_PUBKEY ... / is not signed)`  
   - 确认使用了正确的密钥 `(--gpg-key)`，建议 `--source` 参数中不要使用 `signed-by`（仓库行中），脚本会自行处理。
   - 如果使用的是密钥文件，确保密钥可读且路径正确。
3. 网络/证书问题 `(Unable to establish SSL connection 或 Failed to fetch)`  
使用 `curl -v <URL>` 或 `wget --spider <URL>` 验证连通性。

## 交叉编译

为提升开发和调试效率，用户可以在 Ubuntu PC 上安装工具链，或使用 [NVIDIA 交叉编译工具链](https://developer.nvidia.com/embedded/jetson-linux-archive) 进行开发和编译。  
编译完成后，可通过 SSH 将可执行文件推送至 Manifold 3 进行调试。  
调试通过后，可将应用打包为 DPK 格式，便于在更多设备上安装和使用。

::: warning 注意：

妙算3设备内置 GCC 版本为9.4.0，若安装的交叉编译工具链高于此版本，可能会引入潜在的不兼容问题。
:::


### 交叉编译构建 PSDK 可执行文件
下载所需 PSDK 版本，修改 `CMakeLists.txt` 文件:

```bash
add_subdirectory(samples/sample_c/platform/linux/manifold2)
add_subdirectory(samples/sample_c++/platform/linux/manifold2)
```

修改为：

```bash
add_subdirectory(samples/sample_c/platform/linux/manifold3)
add_subdirectory(samples/sample_c++/platform/linux/manifold3)
```

在 PSDK 文件夹内打开终端，输入：

```bash
mkdir build && cd build
cmake .. -DCMAKE_C_COMPILER=aarch64-linux-gnu-gcc -DCMAKE_CXX_COMPILER=aarch64-linux-gnu-g++
make
```
编译完成后，bin 目录下将生成可在 Manifold 3 上运行的可执行文件   `dji_sdk_demo_on_manifold3 `、 `dji_sdk_demo_on_manifold3_cxx `。
