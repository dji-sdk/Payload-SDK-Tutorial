# 使用E-Port V2开发套件

E-Port V2 开发套件方便开发者快速启动基于 E-Port V2 接口的PSDK负载开发，到手开箱即用。可将E-Port V2接口转接成常见的通信接口和电源接口，方便开发调试。
E-Port V2 开发板内置MCU，MCU型号：GD32F527ZMT7，外扩SDRAM 8MB，MCU器件规格和技术文档（可从GD32官网获取）。
通过开关（图示11）可以切换连接到外置设备，需将外置设备连接到USB接口（图示12），该USB可连接至第三方MCU或者SOC开发板设备的USB Device接口。若外置设备不支持USB，也可以使用USB-TTL模块（支持 FT232、CP2102芯片）转换成串口。同时也支持USB VCOM协议 ( VID:PID要设置成0x2CA3:0xF002)。

> 最低版本要求：不低于 PSDK 3.13.0

| 2 | E-PortV2 电源开关 | 11 | E-Port V2 USB 切换开关 |

| 9 | MCU SWD 烧录接口 | 18 | MCU Loader 按键 |
|  |  | 19 | 电源指示灯 |

* 电源开关（Enable丝印）：通过拨码ON或者OFF控制飞机电源输入，开关旁边是绿色的电源指示灯。
* 复位按键：左下角丝印为RESET的按键，用于MCU复位。
* 烧录接口：左下角复位按键旁边丝印为SDIO的引脚，支持SWD接口烧录MCU芯片。
* 调试串口：左下角Debug丝印的四个引脚，其中TX/RX分别对应MCU的PA0/PA1引脚。
* 功率电源输出：3个XT30接口，分别能给出5V/12V/VCC电压的功率输出。
* Loader按键：右上角丝印为Loader，可以在复位时强制进入MCU的Loader进行升级（需要烧录配套的Loader固件）
* Switch开关：USB信号选择开关，当开关切换到Onboard时，Eport口的USB 2.0信号连通到MCU，此时作为MCU可作为开发板开发PSDK设备；当开关切换到Expand端，Eport口的USB 2.0/3.0信号可以连通到Type-A口上，此时可以作为转接板，接入外部的PSDK设备进行二次开发。
* USB接口：飞机作为Host，板载MCU/扩展外部设备的USB作为Device。
* 扩展IO：在调试串口旁边有一排扩展IO，引脚定义如下，可结合实际情况灵活使用。

| 3V3 | 5V | SCK | NSS | IIC-SCL | IIC-SDA | RX2 | TX2 | PA5 | PA3 |
| - | - | - | - | - | - | - |- | - | - |
| - | - | PC10 | PA15 | PA8 | PC9 | PB11 | PB10 | PA5 | PA3 |
| **GND** | **GND** | **MISO** | **MISO** | **CAN-TX** | **CAN-RX** | **RX0** | **TX0** | **PA6** | **PA4** |
| - | - | PC11 | PC12 | PA12 | PA11 | PA10 | PA9 | PA6 | PA4 |

> **注意**
> * EPORT 2.0开发板和用户设备的USB接口之间的线缆建议小于25cm，以保证USB信号质量。

# Skyport V3 转接环
相比Skyport V2，Skyport V3仅支持M400飞行平台，不兼容M300和M350。内置DJI SDK认证芯片，主要功能是实现云台快拆（保证了接口连接的可靠性和稳定性），且将E-Port V2接口转换成同轴线接口（线序定义参考M400硬件连接部分），方便用户开发和适配自定义负载。

# 快速开始

> **提示**
> * 您可以直接使用[PSDK_EPORT_V2_GD32_LOADER_APP.bin](https://terra-1-g.djicdn.com/71a7d383e71a4fb8887a310eb746b47f/osdk/PSDK_EPORT_V2_GD32_LOADER_APP.bin)（整合了bootloader以及app），烧录到板载MCU的**0x08000000**地址上，快速体验PSDK应用的控件功能以及本地升级功能。

## 使用板载MCU开发板

1. 将E-Port V2开发板的ENABLE开关拨到"ON"，SWITCH开关拨到 "ONBOARD"端。接入到飞机的E-Port V2接口，目前推荐使用1号口。
2. 连接Jlink调试器（SWD接口）和USB转串口到对应开发板的排针上，进行开发调试前的准备工作。
3. 无人机整机启动，检查E-Port V2开发板绿色电源指示灯是否亮起。
4. 编译PSDK GD32F527的Sample工程，路径为：`samples/sample_c/platform/rtos_freertos/gd32f527_development_board/project`
5. 首先使用MDK编译bootloader工程，编译成功后，进行烧录。(MDK需要安装GD32的[MDK Pack](https://www.gd32mcu.com/cn/download/7?kw=GD32F5)）

7. 最后，观察PSDK调试串口的日志是否输出正常。



## 使用外部设备（如树莓派4B）
1. 将E-Port V2开发板的ENABLE开关拨到"ON"，SWITCH开关拨到 “EXPAND”端。使用EPORT V2同轴线连接飞机的E-Port V2接口，目前推荐使用1号口。
2. 将树莓派4B开发板烧录好官方镜像（使用x64位）后，通过USB线连接树莓派4B开发板的供电接口（包含USB 2.0的通讯能力）。
3. 无人机整机启动，检查E-Port V2开发板绿色电源指示灯是否亮起，树莓派是否启动正常。
4. 搭建树莓派软件开发环境部分请参考本教程文档[快速入门]-[E-Port到手即用]-[使用树莓派开发套件]章节。

特别说明，当使用E-Port V2开发PSDK，并且PSDK链路配置为`DJI_USE_ONLY_UART`，可使用的VCOM、USB串口组要在下列配置中选择：
> * CP2101作为主芯片的USB串口，符合USB 2.0协议，VID:PID为10C4:EA60
> * FT232T作为主芯片的USB串口，符合USB 2.0协议，VID:PID为0403:6001
> * 将外部设备实现USB VCOM，VID:PID设置为2CA3:F002
