## 简介

本文通过一个基于 **E‑Port V2 开发套件**的快速体验 Demo，帮助开发者在最短时间内完成从负载硬件接入、PSDK 示例配置到 Demo 运行验证的完整流程。  
通过该 Demo，开发者可以快速了解**基于 PSDK 的负载开发方式**，并掌握负载设备如何通过 E‑Port V2 接口与飞行平台及 DJI Pilot 建立通信与交互。

## 通过本 Demo 你将完成什么
完成本快速体验 Demo 后，你将能够：
- 使用 E‑Port V2 开发板作为负载设备，与飞行平台进行硬件连接。  
- 在 GD32F527 板载 MCU 上运行 PSDK 示例程序。  
- 成功运行负载 Demo 程序，并与飞行平台建立通信。  
- 在 DJI Pilot 上验证负载被正确识别，并完成基础交互验证。  

## E‑Port V2 开发套件介绍
E-Port V2 开发套件方便开发者快速启动基于 E-Port V2 接口的 PSDK 负载开发，到手开箱即用。可将 E-Port V2接口转接成常见的通信接口和电源接口，方便开发调试。  
E-Port V2 开发板内置 MCU，MCU 型号：GD32F527ZMT7，外扩 SDRAM 8MB，MCU 器件规格和技术文档（可从 GD32官网获取）。  
通过开关（图示11）可以切换连接到外置设备，需将外置设备连接到 USB 接口（图示12），该 USB 可连接至第三方 MCU 或者 SOC 开发板设备的 USB Device 接口。若外置设备不支持 USB，也可以使用 USB-TTL 模块（支持 FT232、CP2102芯片）转换成串口。同时也支持 USB VCOM 协议 ( VID:PID 要设置成0x2CA3:0xF002)。
:::tip 说明：
最低版本要求：不低于 PSDK 3.13.0
:::

* 电源开关（Enable 丝印）：通过拨码 ON 或者 OFF 控制飞机电源输入，开关旁边是绿色的电源指示灯。
* 复位按键：左下角丝印为 RESET 的按键，用于 MCU 复位。
* 烧录接口：左下角复位按键旁边丝印为 SDIO 的引脚，支持 SWD 接口烧录 MCU 芯片。
* 调试串口：左下角 Debug 丝印的四个引脚，其中 TX/RX 分别对应 MCU 的 PA0/PA1引脚。
* 功率电源输出：3个 XT30接口，分别能给出5V/12V/VCC 电压的功率输出。
* Loader 按键：右上角丝印为 Loader，可以在复位时强制进入 MCU 的 Loader 进行升级（需要烧录配套的 Loader 固件）
* Switch 开关：USB 信号选择开关，当开关切换到 Onboard 时，Eport 口的 USB 2.0信号连通到 MCU，此时作为 MCU 可作为开发板开发 PSDK 设备；当开关切换到 Expand 端，Eport 口的 USB 2.0/3.0信号可以连通到 Type-A 口上，此时可以作为转接板，接入外部的 PSDK 设备进行二次开发。
* USB 接口：飞机作为 Host，板载 MCU/扩展外部设备的 USB 作为 Device。
* 扩展 IO：在调试串口旁边有一排扩展 IO，引脚定义如下，可结合实际情况灵活使用。

### 使用板载 MCU 开发板
:::tip 提示:
您可以直接使用 [PSDK_EPORT_V2_GD32_LOADER_APP.bin](https://terra-1-g.djicdn.com/71a7d383e71a4fb8887a310eb746b47f/osdk/PSDK_EPORT_V2_GD32_LOADER_APP.bin)（整合了 bootloader 以及 app），烧录到板载 MCU 的**0x08000000**地址上，快速体验 PSDK 应用的控件功能以及本地升级功能。
:::

1. 将 E-Port V2开发板的 ENABLE 开关拨到"ON"，SWITCH 开关拨到 "ONBOARD"端。接入到飞机的 E-Port V2接口，目前推荐使用1号口。
2. 连接 Jlink 调试器（SWD 接口）和 USB 转串口到对应开发板的排针上，进行开发调试前的准备工作。
3. 无人机整机启动，检查 E-Port V2开发板绿色电源指示灯是否亮起。



## 开发环境准备

1. 安装 armcc-cortex-m33 工具链。
2. 安装 [Keil MDK](http://www2.keil.com/mdk5/) 和相关开发工具
    * C Compiler:  Armcc.exe V5.06 update 6 (build 750)
    * Assembler:              Armasm.exe V5.06 update 6 (build 750)
    * Linker/Locator:         ArmLink.exe V5.06 update 6 (build 750)
    * Library Manager:        ArmAr.exe V5.06 update 6 (build 750)
    * Hex Converter:          FromElf.exe V5.06 update 6 (build 750)

3. 使用 Keil 软件
激活 keil MDK 软件后，在 [兆易创新官网](https://www.gd32mcu.com/cn/download?kw=f527&lan=cn)  手动下载最新的 GD32F527 AddOn 包，并安装 GigaDevice.GD32F5xx_DFP.1.3.0.pack 驱动包。

## 运行 Demo
最新的 [DJI PSDK 软件开发包](https://github.com/dji-sdk/Payload-SDK/releases) 中包含着应用程序的示例代码，可以通过如下步骤编译运行示例程序，来快速了解和体验 PSDK 相关功能。
### 烧录 Bootloader
:::tip 说明：
Bootloader 仅在首次使用开发板或需要升级时烧录一次，后续开发只需烧录应用程序。
:::
1. 编译 PSDK GD32F527的 Sample 工程，路径为：`samples/sample_c/platform/rtos_freertos/gd32f527_development_board/project`
2. 首先使用 MDK 编译 bootloader 工程，编译成功后，进行烧录。（MDK 需要安装 GD32的 [MDK Pack](https://www.gd32mcu.com/cn/download/7?kw=GD32F5)）

### 补充应用信息
将获取的[应用信息](#获取应用信息)，填入  `sample/sample_c/platform/rtos_freertos/application/dji_sdk_app_info.h` 文件中。
```
#define USER_APP_NAME               "your_app_name"
#define USER_APP_ID                 "your_app_id"
#define USER_APP_KEY                "your_app_key"
#define USER_APP_LICENSE            "your_app_license"
#define USER_DEVELOPER_ACCOUNT      "your_developer_account"
#define USER_BAUD_RATE              "460800"
```

### 编译并烧录
* 使用 Keil MDK IDE 编译示例代码为示例程序。
* 编译示例代码后，将编译后的程序**烧录**到负载设备中（如 GD32F527 开发板）。
* 如需调试示例程序，请将串口调试工具的波特率设置为：`921600`。

若同时满足以下条件，则说明 Demo 运行成功：
- 串口日志中显示 PSDK 初始化成功
- DJI Pilot 上成功识别负载并显示自定义控件
