This document aims to explain how to select the operating system and development platform for payload equipment based on the chosen flight platform. Selection requires comprehensive consideration of factors such as differences in PSDK functionality support across platforms and systems, resource consumption of payload programs, and the toolchain supported by PSDK.

- Stack: 12288 bytes
- Heap: 40960 bytes
- Text: 755359 bytes
- Data: 3872 bytes
- Bss: 23848 bytes

- CPU: 30%

## Select development platform
### Use Manifold 3
Manifold 3 comes pre-installed with the software environment required for PSDK development, allowing users to develop applications directly on the device. Alternatively, users can set up a cross-compilation environment on their PC by downloading the [NVIDIA cross-compilation toolchain](https://developer.nvidia.com/embedded/jetson-linux-archive) or by manually installing the gcc-aarch64-linux-gnu and g++-aarch64-linux-gnu cross-compilation toolchains.

### Using third-party development platforms
PSDK supports the use of the following tools to compile load devices developed based on PSDK. During development, please select the matching toolchain from the table according to your chosen **development platform**.

For the detailed description of cross-platform porting, please refer to [Cross-platform porting](https://developer.dji.com/doc/payload-sdk-tutorial/en/payload-quick-start/porting.html).
