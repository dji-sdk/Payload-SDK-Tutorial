# E-Port V2 Development Kit

The E-Port V2 Development Kit allows developers to quickly start payload development based on Matrice 400. The E-Port V2 port can be converted into common communication ports and power ports, facilitating development and debugging.

The E-Port V2 development board features a GD32F527ZMT7 MCU and 8 MB of external SDRAM. Detailed device specifications and technical documentation are available on the GD32 official website.

By using the switch (Callout 11), you can connect to external devices. The external device needs to be connected to the USB port (Callout 12), which can be connected to the USB device port of third-party MCU or SOC development boards. If the external device does not support USB, you can use a USB-TTL module (supporting FT232, CP2102 chips) to convert it into a serial port. It also supports the USB VCOM protocol (VID:PID should be set to 0x2CA3:0xF002).

> Minimum version requirement: no less than PSDK 3.13.0

| 1 | E-Port V2 Connector | 10 | MCU Reset Button |
| 2 | E-Port V2 Power Switch | 11 | E-Port V2 USB Toggle Switch |

| 4 | MCU UART Port | 13 | E-Port V2 Signal Port |
| 5 | MCU IIC/CAN Port | 14 | MCU Indicator |
| 6 | MCU SPI Port | 15 | XT30 Power Output Port (5V) |

| 8 | MCU UART Debugging Port | 17 | XT30 Power Output Port (VCC) |
| 9 | MCU SWD Programming Port | 18 | MCU Loader Button |
|  |  | 19 | Power Indicator |

* Power Switch (Enable Silkscreen): Controls aircraft power input by toggling ON or OFF, with a green power indicator next to the switch.
* Reset Button: The button with the RESET silkscreen in the lower left corner, used for MCU reset.
* Programming Port: The pin with the SDIO silkscreen next to the reset button in the lower left corner, supports SWD port programming for MCU chips.
* Debug Serial Port: The four pins with the Debug silkscreen in the lower left corner, where TX/RX correspond to the MCU's PA0/PA1 pins respectively.
* Power Supply Output: Three XT30 ports, each providing power output at 5 V, 12 V, and VCC voltage.
* Loader Button: The top right silkscreen is Loader, which can force the MCU into the Loader for upgrades during reset (requires burning the corresponding Loader firmware).
* Switch: USB signal selection switch. When the switch is set to Onboard, the USB 2.0 signal of the E-Port is connected to the MCU, allowing the MCU to be used as a development board for PSDK devices. When the switch is set to Expand, the USB 2.0/3.0 signal of the E-Port can be connected to the Type-A port, allowing it to be used as an adapter board to connect external PSDK devices for secondary development.
* USB Port: The aircraft acts as the host, with the onboard MCU/external devices' USB acting as the device.
* Expansion IO: Next to the debug serial port is a row of expansion IOs, with pin definitions as follows, which can be used flexibly according to actual needs.

| 3V3 | 5V | SCK | NSS | IIC-SCL | IIC-SDA | RX2 | TX2 | PA5 | PA3 |
| - | - | - | - | - | - | - | - | - | - |
| - | - | PC10 | PA15 | PA8 | PC9 | PB11 | PB10 | PA5 | PA3 |
| **GND** | **GND** | **MISO** | **MISO** | **CAN-TX** | **CAN-RX** | **RX0** | **TX0** | **PA6** | **PA4** |
| - | - | PC11 | PC12 | PA12 | PA11 | PA10 | PA9 | PA6 | PA4 |

> **Note**
> * It is recommended that the cable length between the EPORT 2.0 development board and the user device's USB interface be less than 25 cm to ensure USB signal quality.

# Skyport V3 Adapter Ring

Compared to Skyport V2, Skyport V3 only supports the Matrice 400 flight platform and is not compatible with Matrice 300 and Matrice 350. Built-in DJI SDK certified chip, the main function is to achieve quick gimbal release (ensuring the reliability and stability of the port connection) and convert the E-Port V2 port into a Round Ribbon Cable port (refer to the Matrice 400 hardware connection section for wire sequence definition), facilitating user development and adaptation of custom payloads.

# Quick Start

> **Tips**
> * You can directly use [PSDK_EPORT_V2_GD32_LOADER_APP.bin](https://terra-1-g.djicdn.com/71a7d383e71a4fb8887a310eb746b47f/osdk/PSDK_EPORT_V2_GD32_LOADER_APP.bin) (which integrates both the bootloader and the app) and flash it to the onboard MCU at address **0x08000000** to quickly experience the widget functions of the PSDK application and its local upgrade capability.

## Using Onboard MCU Development Board

1. Switch the ENABLE switch of the E-Port 2.0 development board to "ON," and the Switch to the "ONBOARD" side. Connect to the aircraft's E-Port 2.0 interface. Currently it is recommended to use Port 1.
2. Connect the Jlink debugger (SWD port) and USB to serial port to the corresponding development board pins for preparation before development and debugging.
3. Power on the aircraft and check if the green power indicator on the E-Port 2.0 development board is on.
4. Compile the PSDK GD32F527 sample project located at: `samples/sample_c/platform/rtos_freertos/gd32f527_development_board/project`.
5. First, use MDK to compile the bootloader project. After successful compilation, proceed with flashing. (MDK requires the [GD32 MDK Pack](https://www.gd32mcu.com/en/download/7?kw=GD32F5) to be installed)

## Use External Devices (Such as RPI-4B)

1. Set the ENABLE switch on the E-Port V2 development board to "ON," and set the SWITCH to the "EXPAND" position. Use a coaxial cable to connect the E-Port V2 to the E-Port V2 port; currently, it is recommended to use port 1.
2. After flashing the official image (using x64 bit) on the Raspberry Pi 4B development board, connect it to the power port of the Raspberry Pi 4B development board (which includes USB 2.0 communication capability) via a USB cable.
3. Power on the aircraft, check if the green power indicator on the E-Port 2.0 development board is on, and if the Raspberry Pi is starting up normally.
4. Refer to the xxxxxx section of this tutorial document for setting up the Raspberry Pi software development environment.
