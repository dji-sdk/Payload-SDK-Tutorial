## Introduction
This article provides a quick-start demo based on the **E‑Port V2 Development Kit**, helping developers complete the entire process—from payload hardware integration and PSDK sample configuration to demo execution and verification—in the shortest possible time.  
Through this demo, developers can quickly understand **PSDK-based payload development** and learn how a payload device communicates and interacts with the flight platform and **DJI Pilot** via the E‑Port V2 port.

## What You Will Achieve with This Demo
After completing this quick-start demo, you will be able to:
- Use the **E‑Port V2 development board** as a payload device and establish a hardware connection with the flight platform.  
- Run PSDK sample programs on the **GD32F527 onboard MCU**.  
- Successfully run the payload demo program and establish communication with the flight platform.  
- Verify in **DJI Pilot** that the payload is correctly recognized and complete basic interaction validation.

## E‑Port V2 Development Kit Overview

The E-Port V2 Development Kit allows developers to quickly start payload development based on Matrice 400. The E-Port V2 port can be converted into common communication ports and power ports, facilitating development and debugging.

The E-Port V2 development board features a GD32F527ZMT7 MCU and 8 MB of external SDRAM. Detailed device specifications and technical documentation are available on the GD32 official website.

By using the switch (Callout 11), you can connect to external devices. The external device needs to be connected to the USB port (Callout 12), which can be connected to the USB device port of third-party MCU or SOC development boards. If the external device does not support USB, you can use a USB-TTL module (supporting FT232, CP2102 chips) to convert it into a serial port. It also supports the USB VCOM protocol (VID:PID should be set to 0x2CA3:0xF002).

:::tip
Minimum version requirement: no less than PSDK 3.13.0
:::

* Power Switch (Enable Silkscreen): Controls aircraft power input by toggling ON or OFF, with a green power indicator next to the switch.
* Reset Button: The button with the RESET silkscreen in the lower left corner, used for MCU reset.
* Programming Port: The pin with the SDIO silkscreen next to the reset button in the lower left corner, supports SWD port programming for MCU chips.
* Debug Serial Port: The four pins with the Debug silkscreen in the lower left corner, where TX/RX correspond to the MCU's PA0/PA1 pins respectively.
* Power Supply Output: Three XT30 ports, each providing power output at 5 V, 12 V, and VCC voltage.
* Loader Button: The top right silkscreen is Loader, which can force the MCU into the Loader for upgrades during reset (requires burning the corresponding Loader firmware).
* Switch: USB signal selection switch. When the switch is set to Onboard, the USB 2.0 signal of the E-Port is connected to the MCU, allowing the MCU to be used as a development board for PSDK devices. When the switch is set to Expand, the USB 2.0/3.0 signal of the E-Port can be connected to the Type-A port, allowing it to be used as an adapter board to connect external PSDK devices for secondary development.
* USB Port: The aircraft acts as the host, with the onboard MCU/external devices' USB acting as the device.
* Expansion IO: Next to the debug serial port is a row of expansion IOs, with pin definitions as follows, which can be used flexibly according to actual needs.


## Hardware Environment Preparation

### Hardware List
- Matrice 400 aircraft  
- E‑Port V2 coaxial cable  
- E‑Port V2 development board  

### Matrice 400 Device Connection
Use the E‑Port V2 coaxial cable to connect the E‑Port V2 development board to the E‑Port V2 port on the Matrice 400.

After completing the connection as shown above, developers can perform PSDK development directly on the onboard **GD32 MCU** of the E‑Port V2 development board. Alternatively, developers may use the USB port on the E‑Port V2 development board to connect an external third‑party development board. The following instructions describe development using the onboard MCU as an example.

### Using the Onboard MCU
:::tip Tip:
You can directly flash [PSDK_EPORT_V2_GD32_LOADER_APP.bin](https://terra-1-g.djicdn.com/71a7d383e71a4fb8887a310eb746b47f/osdk/PSDK_EPORT_V2_GD32_LOADER_APP.bin) (which integrates both the bootloader and application) to address **0x08000000** of the onboard MCU to quickly experience PSDK widget functions and local upgrade features.
:::

1. Set the **ENABLE** switch on the E‑Port V2 development board to **ON**, and set the **SWITCH** to the **ONBOARD** position. Connect the board to the aircraft’s E‑Port V2 port. It is currently recommended to use **Port 1**.
2. Connect a **J-Link debugger** (SWD port) and a **USB‑to‑serial adapter** to the corresponding pin headers on the development board to prepare for debugging.
3. Power on the aircraft and check whether the green power indicator LED on the E‑Port V2 development board is illuminated.

## Development Environment Preparation

1. Install the **armcc-cortex-m33** toolchain.
2. Install [Keil MDK](http://www2.keil.com/mdk5/) and the related development tools:
   * C Compiler: Armcc.exe V5.06 update 6 (build 750)  
   * Assembler: Armasm.exe V5.06 update 6 (build 750)  
   * Linker/Locator: ArmLink.exe V5.06 update 6 (build 750)  
   * Library Manager: ArmAr.exe V5.06 update 6 (build 750)  
   * Hex Converter: FromElf.exe V5.06 update 6 (build 750)  
3. Using Keil software, after activating Keil MDK, manually download the latest **GD32F527 AddOn package** from the [GigaDevice official website](https://www.gd32mcu.com/en/download?kw=f527&lan=en&submit=) and install the **GigaDevice.GD32F5xx_DFP.1.3.0.pack** device support package.

## Run the Demo

The latest [DJI PSDK Software Development Kit](https://github.com/dji-sdk/Payload-SDK/releases) includes sample application code.  
You can compile and run the sample programs using the steps below to quickly understand and experience PSDK-related features.

### Flashing the Bootloader
:::tip Note:
The bootloader only needs to be flashed once when the development board is used for the first time or when an upgrade is required. For subsequent development, only the application needs to be flashed.
:::

1. Compile the PSDK GD32F527 sample project located at:  
   `samples/sample_c/platform/rtos_freertos/gd32f527_development_board/project`
2. First, use **Keil MDK** to build the bootloader project. After successful compilation, flash the bootloader to the board.  (Keil MDK requires the installation of the GD32 [MDK Pack](https://www.gd32mcu.com/en/download?kw=GD32F5&lan=en&submit=).)

### Completing Application Information
Enter the obtained [application information](#obtain-application-information) into the `sample/sample_c/platform/rtos_freertos/application/dji_sdk_app_info.h` file; otherwise, the sample application will not run properly.
```
#define USER_APP_NAME               "your_app_name"
#define USER_APP_ID                 "your_app_id"
#define USER_APP_KEY                "your_app_key"
#define USER_APP_LICENSE            "your_app_license"
#define USER_DEVELOPER_ACCOUNT      "your_developer_account"
#define USER_BAUD_RATE              "460800"
```


### Build and Flash
- Use the **Keil MDK IDE** to compile the sample code into an executable application.
- After compilation, **flash** the generated program to the payload device (such as the GD32F527 development board).
- If you need to debug the sample application, set the baud rate of the serial debugging tool to **`921600`**.

### Demo Runtime Verification
If the following conditions are met, the demo is considered to be running successfully:
- The serial log indicates that PSDK initialization is successful.
- The payload is successfully recognized in **DJI Pilot**, and custom widgets are displayed.
