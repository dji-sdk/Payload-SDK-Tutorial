## Matrice 400 Device Connection

Before starting the device connection for Matrice 400, read the Matrice 400 User Manual first, especially the Overview section section, and familiarize yourself with the introduction to the aircraft components.

The Matrice 400 User Manual has been updated with FOV information. Specifically, you can refer to the "FOV" section in the [Matrice_M400 User Manual V1.02](https://dl.djicdn.com/downloads/Matrice_400/UM/20250908/Matrice_400_User_Manual_v1.02_zh-cn.pdf).

### Connecting Devices via the E-Port V2 Port

There are four E-Port V2 ports located at the bottom of Matrice 400, as shown in the figure below (you can identify them by comparing with the actual device):

<div> <div align=center> <img src="https://terra-1-g.djicdn.com/71a7d383e71a4fb8887a310eb746b47f/psdk/M400/eport_v2_num.png" style="width:400px"/> </div> <div style="text-align: center"><p>Figure. Matrice 400 E-Port V2 Port Numbering</p></div> </div>

There are three ways to connect devices to Matrice 400 via the E-Port V2 port:
1. X-Port Standard Gimbal (connects to E-Port V2 using the X-Port development board; requires connecting the development board's MCU serial port via a DuPont cable).
2. Skyport V3 adapter ring and Skyport V3 coaxial cable.
3. E-Port V2 coaxial cable.

The connection diagram is as follows:

<div> <div align=center> <img src="https://terra-1-g.djicdn.com/71a7d383e71a4fb8887a310eb746b47f/psdk/M400/%E5%9B%BE%E7%89%872.jpg" style="width:auto"/> </div> <div style="text-align: center"><p>Diagram. Matrice 400 E-Port V2 Connection Methods</p></div> </div>

After completing the connection as shown above, users can directly develop with the PSDK using the GD32 microcontroller built into the E-Port V2 development board, or they can use the USB port on the E-Port V2 development board to connect to a third-party development board for development. If connecting the X-port development board to the E-Port V2, DuPont wires are required to connect the MCU serial port of the development board.

The No. 4 E-Port V2 port on Matrice 400 supports expansion via a connected hub. Refer to the [E-Port V2 HUB Reference Schematic](https://terra-1-g.djicdn.com/71a7d383e71a4fb8887a310eb746b47f/psdk/M400/hub%E5%8F%82%E8%80%83%E5%8E%9F%E7%90%86%E5%9B%BE%E6%9B%B4%E6%96%B0.pdf) to design a secondary hub, which allows you to expand up to four additional E-Port V2 ports through the hub. The connection method for the expanded E-Port V2 ports is the same as for the native ports.

### Connect Development Board to E-Port
<div>
<div align=center>
<img src="https://terra-1-g.djicdn.com/71a7d383e71a4fb8887a310eb746b47f/psdk/quick%20demo/6-2.%20EA230%20%E5%9F%BA%E4%BA%8EE-Port%20%E6%8E%A5%E5%8F%A3%E8%BF%9E%E6%8E%A5%E8%AE%BE%E5%A4%87-%E8%8B%B1%E6%96%87.png" style="width:70%"/>

2. Connect the male end of the coaxial cable to the E-Port connector with side A facing the bottom of the E-Port adapter board (foam side) and side B facing the front of the E-Port (component side).

  ::warning Warning:
 The E-Port coaxial USB-C cable doesn't have a foolproof design, allowing A/B side to be reversibly connected. :::

3. Switch the USB device/host mode to "Host".
4. If powering through the E-Port Development Kit, ensure check the voltage values for each port to prevent damaging the payload device.
5. Connect to the payload device's serial port.
6. Connect the payload device USB device/USB OTG/network port to the USB 2.0 interface.

  :::tip Note:

:::

7. Turn the E-Port dip switch to "ON".

## Hardware Reference Design

[Hardware Reference Design](https://github.com/dji-sdk/Payload-SDK/blob/master/doc/reference_designs/E-Port%20Schematic%20Reference.pdf)

The above hardware connection is mainly based on DJI's development accessories to build a development environment, facilitating developers to quickly build a hardware environment and start PSDK software development. Usually, when the device is developed, it can be directly integrated into the device hardware environment based on the hardware pins on the aircraft, without the need for DJI expansion accessories, such as OSDK expansion adapter boards, E-PORT kits, etc.

To facilitate developers in integrating hardware connections based on OSDK/PSDK interfaces, DJI provides reference schematics for the Matrice 300 RTK OSDK interface, Matrice 350 RTK E-Port interface, and Matrice 30/Matrice 30T gimbal port. Developers can use these as references and adapt them to their own hardware designs.
