## Introduction

DJI SDK Certified Chip (DJI SDK CC for short) is used to certify third-party payload devices developed based on DJI Eport/Eport-Lite interfaces, ensuring that the devices can work properly and communicate effectively with drones, playing the role of payload security authentication and function authorization.

To this end, PSDK provides a highly integrated software interface. After users integrate the DJI SDK certified chip into their own development platform according to hardware requirements, they do not need to worry about the complex protocol interaction and security algorithm of the certified chip. They only need to register the standardized I2C device operation abstract interface to quickly integrate the certified chip into their own devices and complete normal authentication and communication with the aircraft.
:::tip Notes:
- Supported models: Matrice 4E/4T and subsequent models. If the device **does not integrate the SDK certified chip, the PSDK function will not be available**.

- Minimum version requirement: no less than PSDK 3.10.0
:::
## Basic information

Specifications of the certified chip:

* Operating voltage range: 1.62 V - 5.5 V

* Operating temperature range: -40° C - 85° C

* Package size: DFN8 package (2 x 3 mm)

* Communication interface: I2C interface (address: 0x2A)

* Communication frequency: up to 400Khz

The following figure shows the pin configuration and connection relationship of the DJI SDK certified chip. The main connections are as follows:

* VCC: power input pin
* GND: ground pin
* NRST: reset pin, external reset circuit or signal
* I2C_SCL: I²C bus interface pin, used for clock signal transmission
* I2C_SDA: I²C bus interface pin, used for data transmission

:::tip Notes:

## Instructions

#### Linux platform (Raspberry Pi 4B)

The following will take the Sample of the Raspberry Pi 4B platform as an example to describe how to quickly integrate the DJI SDK on the Linux platform Authentication chip and complete application binding.

1. Transfer the DJI SDK authentication chip through the adapter board, power the authentication chip through the extended IO pins of the Raspberry Pi 4B development board, and connect the IIC communication and RST reset pins. The connection line sequence is shown in the figure below. The sample code uses the I2C device `/dev/i2c-1`.

- SCL: GPIO3
- SDA: GPIO2
- RST: GPIO4

Recommended integration method for encryption chip:
* Debugging stage: Use tweezers to place the authentication chip in the corresponding burning adapter (05D4 DFN8 2x3, it is recommended to purchase it on the e-commerce platform), and connect the development board through the DuPont line for debugging.
* Mass production stage: Add the authentication chip to the hardware PCB design and solder it to the board.

:::warning Warning:
 You can use the `i2cdetect` tool to scan the connected I2C device to quickly verify whether the authentication chip is working properly. If it cannot be identified, you can appropriately reduce the I2C frequency and try again.
:::
1. The user registers the implementation of the standard I2C device driver by calling the `DjiPlatform_RegHalI2cHandler` software interface. When the PSDK is initialized, the authentication process of the authentication chip is automatically completed through the registration callback. For detailed implementation, please refer to `samples/sample_c/platform/linux/raspberry_pi/hal/hal_i2c.c`.

```
returnCode = DjiPlatform_RegHalI2cHandler(&i2CHandler);
if (returnCode != DJI_ERROR_SYSTEM_MODULE_CODE_SUCCESS) {
printf("register hal i2c handler error");
return DJI_ERROR_SYSTEM_MODULE_CODE_SYSTEM_ERROR;
}
```

3. After successful compilation, run the PSDK program. If the authentication chip hardware connection is normal, the SN of the authentication chip can be successfully obtained. At this time, the log will prompt "SDK accessories are not bound".

4. Use DA2 to log in to the developer account, enter the Payload SDK tab on DA2, and observe whether the PSDK device can be recognized normally. After successful recognition, click the bind button to perform the binding operation.

5. After successful binding, the PSDK device can run normally.
