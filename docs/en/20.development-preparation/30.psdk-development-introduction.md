## What is PSDK Development?
PSDK (Payload SDK) is a development kit provided by DJI for third-party developers to extend the functionality of DJI flight platforms. With PSDK, developers can deeply integrate their own hardware or intelligent algorithms into drone systems, enabling capabilities such as data acquisition, intelligent analysis, and automatic control. PSDK enables developers to:
- Develop hardware payload devices that can be mounted on drones
- Develop intelligent software applications running on onboard computing platforms
- Access flight status and sensor data in depth, and control systems such as gimbals and payloads

## PSDK Software Architecture Overview
```
Application Layer (developer business code)
      ↓
PSDK Core Layer (communication / device / data management)
      ↓
Platform Layer (payload platform / Manifold platform)
```
- Application Layer: Implements business logic, such as algorithms, control strategies, and data processing.
- PSDK Core Layer: Responsible for device management, data communication, and interface encapsulation.
- Platform Layer: Connects to DJI flight platforms or Manifold computing hardware.

## Two Development Approaches
### Payload Development
Payload development refers to using PSDK to develop external physical devices that can be mounted on DJI flight platforms, such as cameras, sensors, mechanical devices, etc.   

**Main Types**
- Payload Customization: Hardware payload products newly designed and developed for the flight platform.
- Payload Adaptation: Porting existing intelligent payloads to new models, or adapting legacy payloads from older aircraft to new platforms.

**Typical Characteristics**  
- Emphasizes hardware design and system integration
- Directly involved in perception and action execution
- Suitable for projects with customized hardware requirements

### Manifold Development  
Manifold development refers to developing and running intelligent software on the onboard Manifold computing platform, enabling data analysis and decision-making capabilities.

**Application Scenarios**  
- AI target recognition and classification
- Real-time data analysis
- Intelligent tracking and decision control  

**Typical Characteristics**  
- Software and algorithms at the core
- Real-time processing using onboard computing power
- Short development cycle and flexible iteration

## Relationship Between Payload and Manifold Development
### 1. Complementary Relationship
- **Payload Development**: Provides the "eyes and hands" for sensing and execution.  
- **Manifold Development**: Provides the "brain" for analysis and decision-making.
- The combination enables a complete intelligent system.

### 2. Collaborative Working Mode
```
Payload Device (e.g., camera)
      ↓ Data acquisition
Manifold Platform (analyze and process)
      ↓ Sends instructions
Payload Device (executes actions)
```
For example: Camera acquires images → Manifold platform performs target recognition → Outputs control instructions → Gimbal or mechanical device executes actions.

### 3. Practical Application Examples
- **Agricultural Inspection**: Multispectral camera payload + disease recognition algorithm
- **Security Patrol**: Thermal imaging payload + personnel detection algorithm
- **Logistics Delivery**: Intelligent cargo hold payload + route planning algorithm

## How to Choose the Development Method?
### Recommendations
- **Mainly hardware innovation** → Payload Development
- **Mainly algorithms and software** → Manifold Development
- **Building a complete intelligent system** → Payload Development + Manifold Development
