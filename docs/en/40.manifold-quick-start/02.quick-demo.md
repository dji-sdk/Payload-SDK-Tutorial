## Introduction
This article guides developers through a complete Manifold development experience step by step using a quick-start demo. It helps developers understand the core concepts and usage workflow in the shortest possible time, and intuitively experience the real value of Manifold development in improving efficiency and enabling business deployment.

## What You Will Achieve with This Demo
After completing this quick-start demo, you will be able to:
- Successfully set up the basic development environment for Manifold 3.
- Log in to Manifold 3 and run the PSDK sample application.
- View PSDK custom widgets in DJI Pilot and perform basic interactions.

If all of the above objectives are achieved, it indicates that you have completed a full quick-start Manifold development experience.

## Hardware Environment Preparation
Manifold 3 provides the following two power supply and connection methods:
1. **Powered by the Aircraft**
   - Matrice 400: E-Port V2 port  
   - Matrice 4D: E-Port port
2. **Powered via a USB Hub PD Port**  
   Connect a power adapter (65W/100W) through a USB hub PD port.

::: tip Notes:
- The USB-C port of Manifold 3 supports connecting a USB hub to set up the development environment (display output is not supported). Recommended models include: UGREEN CM512 Type-C Hub, Anker (A8380) 553 PowerExpand 8-in-1 USB-C Hub.
- For Matrice 4D/4TD series aircraft, when using connection method 1, the wiring between the aircraft and Manifold 3 must be completed before powering on the aircraft. Hot-plugging is not supported with this connection method.
- When connecting to the aircraft via E-Port V2, users can directly connect a PC to the aircraft tuning port using a USB cable and access Manifold 3 via different IP addresses corresponding to different ports through a USB virtual network adapter.
- When using a USB hub connection, users can connect to a local area network via Ethernet and access Manifold 3 by logging in via SSH using the IP address assigned by the router.
- In both connection methods, users can access Manifold 3 via SSH or VNC for development and debugging.
:::

> **Tip**: First-time users are strongly recommended to use the **“USB hub power supply + LAN + SSH login”** method, as it is the simplest path with the fewest potential issues.

## Development Environment Preparation
### Log in to Manifold 3
Manifold 3 comes with the software environment required for PSDK development preinstalled. After logging in, users can start application development directly.

The following two login methods are supported:
- SSH (recommended)
- VNC remote desktop

The following example uses SSH login for illustration.  

**SSH Login Information (Matrice 400)**

| E-Port No. | SSH Address |
| :---- | :---- |
| E1 | 192.168.42.120 |
| E2 | 192.168.42.130 |
| E3 | 192.168.42.140 |

Default login credentials:
- Default username: dji  
- Default password: dji  

### Install the Cross-Compilation Toolchain
To improve development and debugging efficiency, you can install the toolchain on an Ubuntu PC, or use the [NVIDIA cross-compilation toolchain](https://developer.nvidia.com/embedded/jetson-linux-archive) for development and compilation.

::: warning Note:
The GCC version preinstalled on Manifold 3 is 9.4.0. Installing a cross-compilation toolchain with a higher version may introduce potential compatibility issues.
:::

## Run the Demo
The latest [DJI PSDK Software Development Kit](https://github.com/dji-sdk/Payload-SDK/releases) includes sample code for Manifold applications. You can compile and run the sample programs by following the steps below to quickly understand and experience the related Manifold application features.

### Modify the Required Configuration Code
Fill the obtained [application information](#obtain-application-information) into the `dji_sdk_app_info.h` file. Otherwise, the sample application will not run properly.

* samples/sample_c/platform/linux/manifold3/application/dji_sdk_app_info.h
* samples/sample_c++/platform/linux/manifold3/application/dji_sdk_app_info.h

```c
/* Exported constants --------------------------------------------------------*/
// ATTENTION: User must goto https://developer.dji.com/user/apps/#all to create your own dji sdk application, get dji sdk application
// information then fill in the application information here.
#define USER_APP_NAME               "your_app_name"
#define USER_APP_ID                 "your_app_id"
#define USER_APP_KEY                "your_app_key"
#define USER_APP_LICENSE            "your_app_license"
#define USER_DEVELOPER_ACCOUNT      "your_developer_account"
#define USER_BAUD_RATE              "460800"
```
### Compile and Experience the Sample
The PSDK package supports CMake compilation. Follow these steps:


1. Compile using the **cmake** command. Open a terminal in the PSDK directory and run the following commands: 
   
   ```bash
   mkdir build && cd build
   cmake .. -DCMAKE_C_COMPILER=aarch64-linux-gnu-gcc -DCMAKE_CXX_COMPILER=aarch64-linux-gnu-g++
   make
   ```
   After compilation, the following executables will be generated in the `build/`directory:

    - `dji_sdk_demo_on_manifold3`
    - `dji_sdk_demo_on_manifold3_cxx`

2. Run the generated executable file to experience the sample code in action.

   ```c
   cd Payload-SDK/
   mkdir cmake-build-debug && cd cmake-build-debug
   cmake ../  && make -j
   ./bin/dji_sdk_demo_on_manifold3
   ```
