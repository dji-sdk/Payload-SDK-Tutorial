## What is Manifold Application (DPK)?
Manifold Application is software developed for Manifold 3 based on DJI PSDK interfaces, abbreviated as Manifold Application (also referred to as DPK application).

Developers can use DJI's provided packaging tool `build_dpk.sh` through cross-compilation to generate an installation package with `.dpk` suffix (Manifold Application package). Then users can install, update, and uninstall the application via DJI Pilot APP or the built-in application management tool `dji_app_ctl` on Manifold 3 devices. This significantly lowers the user threshold (compared to Manifold 2, users don't need to learn Linux commands, reducing deployment difficulties in field operations).

## How to Develop Manifold Application?
When developing Manifold Application, the following constraints must be followed, and development recommendations should be considered:

### Constraints:
* Third-party dependencies must use static libraries. Dynamic libraries are not supported and may cause install or start app failures.
* When reading/writing configuration files in source code, use relative paths instead of absolute paths to prevent install or start app failures.
* Refer to `samples/sample_c/platform/linux/manifold3/app_json/app.json` in DJI PSDK for creating your own `app.json` file. This file must strictly follow specifications:
  - Developer website-generated APP ID fill in `user_app_id` field
  - Application version(`firmwareVersion`) in code fill in `firmware_version` field
  - Compiled binary fill in `bin` field
  - Configuration files fill in `userconfig` field

Incorrect app.json file may cause the packing xxx.dpk failure or app installation failure. For full specifications, see `samples/sample_c/platform/linux/manifold3/app_json/README.md` or [app.json Configuration Guide](https://stag-developer.dbeta.me/doc/payload-sdk-tutorial/develop/en/manifold-quick-start/build-dpk.html).

### Recommendations:
* Store application logs in `data/logs` directory to ensure app log export and data clear function.
* Store media files in `data/media` directory to maintain media-related features and data management.
