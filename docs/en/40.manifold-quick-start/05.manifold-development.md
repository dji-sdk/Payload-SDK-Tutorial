## Quick Integration of Third-Party Dependencies
To help developers quickly and easily integrate and call third-party dependencies required by the main application in a non-rooted environment, we provide the installation script [install.py](https://terra-1-g.djicdn.com/71a7d383e71a4fb8887a310eb746b47f/psdk/install.py) to swiftly download and install dependencies.

### Script Overview
This script installs software without root privileges (rootless), offering two installation methods: apt and deb.
- apt: Installs packages based on APT repository resolution.
- deb: Installs packages from `.deb` files/URLs.
:::tip Notes:
- When installing via the apt subcommand, custom APT repositories and GPG keys are supported.
- Custom data and database storage directories are supported (default path: `/open_app/install`).
:::
### Requirements
- Python 3.7 or later.
- Internet access (for APT repositories or `key.deb URL` retrieval).

### Usage
- View help:

    ```bash
    python3 install.py -h
    ```

- View help for apt subcommands:

    ```bash
    python3 install.py apt -h
    ```

- View help for deb subcommands:

    ```bash
    python3 install.py deb -h
    ```

Refer to the following format for specific usage commands:  
    `python3 install.py [sub_command（apt/deb）] [params ...] `

### Running Example

During script execution, it will list all software requiring new installation. Software already present in the system or recorded in the user-defined database will not be installed again.

#### APT Installation:

- Use system repositories and specify the installation directory (/open_app/user_install) to install gdebi:

    ```bash
    python3 install.py apt gdebi -d /open_app/user_install
    ```

- Add ROS2 sources and keys, install curl, htop, and ros-foxy-ros-base in the default directory:
  - `curl htop ros-foxy-ros-base`: List of required software packages.
  - `--source “deb https://mirrors.aliyun.com/ros2/ubuntu/ focal main”`: Adds the ROS2 source; the APT source is specified as a string.
  - `--gpg-key “https://raw.githubusercontent.com/ros/rosdistro/master/ros.asc”`: Specifies the APT source key, downloaded via URL.
  
  ```bash
  python3 install.py apt curl htop ros-foxy-ros-base --source "deb https://mirrors.aliyun.com/ros2/ubuntu/ focal main" --gpg-key "https://raw.githubusercontent.com/ros/rosdistro/master/ros.asc"
  ```

- Add multiple sources and keys, installing emqx and ros-foxy-ros-desktop to the default directory:
  - `ros-foxy-desktop emqx-enterprise`: List of software to install.
  - `--source “/open_app/test/emqx_emqx-enterprise5.list”`: Adds the emqx source; the APT source is specified as a file path.
  - `--source “deb https://mirrors.aliyun.com/ros2/ubuntu/ focal main”`: Adds the ROS2 source; the APT source is specified as a string.
  - `--gpg-key /open_app/emqx_emqx-enterprise5-archive-keyring.gpg`: Specifies the key for the APT source, using the file path method to obtain the key.
  - `--gpg-key “https://raw.githubusercontent.com/ros/rosdistro/master/ros.asc”`: Specifies the APT source key, downloaded via URL.

  ```bash
  python3 install.py apt ros-foxy-desktop emqx-enterprise --source "/open_app/test/emqx_emqx-enterprise5.list" --source "deb https://mirrors.aliyun.com/ros2/ubuntu/ focal main" --gpg-key /open_app/emqx_emqx-enterprise5-archive-keyring.gpg --gpg-key "https://raw.githubusercontent.com/ros/rosdistro/master/ros.asc"
  ```

#### deb Installation
File path + URL:

```bash
python3 install.py deb --deb_source “https://dl.min.io/server/minio/release/linux-arm64/minio_20250907161309.0.0_arm64.deb”  --deb_source /open_app/curl_7.68.0-1ubuntu2.25_arm64.deb --dir /open_app/user_install
```

### Troubleshooting
1. Package not found `(N: Unable to locate package <pkg>)`   
Verify the `--source` parameter is correct and the package is indeed present in the source.
2. GPG Error `(NO_PUBKEY ... / is not signed)`  
   - Confirm the correct key is used `(--gpg-key)`. Avoid specifying `signed-by` in the `--source` parameter (in the repository line); the script handles this automatically.
   - If using a key file, ensure the key is readable and the path is correct.
3. Network/certificate issues `(Unable to establish SSL connection or Failed to fetch)`
Verify connectivity using `curl -v <URL>` or `wget --spider <URL>`.

## Cross Compilation

To improve development and debugging efficiency, users can install the toolchain on an Ubuntu PC, or use the [NVIDIA cross-compilation toolchain](https://developer.nvidia.com/embedded/jetson-linux-archive) for development and compilation.  
After compilation, the executable file can be transferred to Manifold 3 via SSH for debugging.  
Once debugging is complete, the application can be packaged into DPK format for easier installation and use on more devices.

::: warning Note:
Manifold 3 has a built-in GCC version of 9.4.0. If the installed cross-compilation toolchain uses a higher GCC version, potential compatibility issues may occur.
:::


### Cross-Compile and Build the PSDK Executable

Download the required PSDK version and modify the `CMakeLists.txt` file:

```bash
add_subdirectory(samples/sample_c/platform/linux/manifold2)
add_subdirectory(samples/sample_c++/platform/linux/manifold2)
```
Change to:
```bash
add_subdirectory(samples/sample_c/platform/linux/manifold3)
add_subdirectory(samples/sample_c++/platform/linux/manifold3)
```

Open a terminal in the PSDK folder and enter:
```bash
mkdir build && cd build
cmake .. -DCMAKE_C_COMPILER=aarch64-linux-gnu-gcc -DCMAKE_CXX_COMPILER=aarch64-linux-gnu-g++
make
```
After compilation, the bin directory will contain the executable files `dji_sdk_demo_on_manifold3` and `dji_sdk_demo_on_manifold3_cxx` that can run on Manifold 3.
