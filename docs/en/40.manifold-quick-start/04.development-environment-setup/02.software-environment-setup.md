

## Firmware Upgrade

**Firmware Upgrade Method:** It is recommended to use DJI Assistant 2 (Enterprise Series) for firmware upgrade and update. For the latest firmware version, refer to the [Release Notes](https://developer.dji.com/doc/payload-sdk-tutorial/en/).

## Accessing Manifold 3

Manifold 3 comes with the software environment required for PSDK development preinstalled. Users can start application development immediately after logging in.

**Ways to access the console or GUI:**
- SSH (recommended, supports Windows Terminal and VS Code)
- VNC remote desktop (GUI)

**The following two methods are supported for file transfer:**
- SCP (WinSCP is recommended for Windows)
- Samba sharing (on Windows, open the shared directory through DJI Assistant 2)

### SSH Login

If a PC is directly connected to the aircraft debugging port using a USB cable, Manifold 3 can be accessed through the following IP addresses depending on its mounting position:

| Flight Platform | Port | SSH Address |
| :-- | :-- | :-- |
| Matrice 4E / 4T / 4TD / 4D | E-Port | 192.168.42.120 |
| Matrice 400 | E1 | 192.168.42.120 |
| Matrice 400 | E2 | 192.168.42.130 |
| Matrice 400 | E3 | 192.168.42.140 |

If a USB Ethernet adapter or Wi‑Fi adapter is connected through the USB docking station of Manifold 3, the IP address can be obtained from the router management page of the connected local network.

**Default login information:**
- Username: `dji`
- Password: `dji`

#### Windows Terminal

Windows 11 users can access Manifold 3 through Windows Terminal. Create a new profile in Windows Terminal and assign it any name you want, for example, `Manifold3`.

Set the command line to:

```bash
ssh dji@192.168.42.140
```

Here, `192.168.42.140` is the IP address of Manifold 3. Save the profile, reopen it, enter the password, and you can access the device.

#### VS Code

To connect using VS Code, open VS Code, select **Remote Explorer**, enter `ssh dji@192.168.42.140` (the IP address of Manifold 3), save it, and connect. After entering the password, you can access Manifold 3 and open related directories in the file explorer.

### SCP File Transfer

After installing WinSCP, click **New Site**, choose **SCP** as the file protocol, enter the IP address of Manifold 3 as the hostname, set the port number to `22`, and use the default username and password `dji`.

## Cross Compilation

To improve development and debugging efficiency, users can install the toolchain on an Ubuntu PC, or use the [NVIDIA cross-compilation toolchain](https://developer.nvidia.com/embedded/jetson-linux-archive) for development and compilation.  
After compilation, the executable file can be transferred to Manifold 3 via SSH for debugging.  
Once debugging is complete, the application can be packaged into DPK format for easier installation and use on more devices.

::: warning Note:
Manifold 3 has a built-in GCC version of 9.4.0. If the installed cross-compilation toolchain uses a higher GCC version, potential compatibility issues may occur.
:::


### Cross-Compile and Build the PSDK Executable

Download the required PSDK version from the <a href="https://github.com/dji-sdk/Payload-SDK/releases" target="_blank">PSDK download page</a>.
Open a terminal in the PSDK folder and enter:
```bash
mkdir build && cd build
cmake .. -DCMAKE_C_COMPILER=aarch64-linux-gnu-gcc -DCMAKE_CXX_COMPILER=aarch64-linux-gnu-g++
make
```
After compilation, the bin directory will contain the executable files `dji_sdk_demo_on_manifold3` and `dji_sdk_demo_on_manifold3_cxx` that can run on Manifold 3.
