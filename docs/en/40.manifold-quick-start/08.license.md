# License Verification Guide

## Brief Description
Add a license management mechanism to the developed application DPK to prevent unauthorized device use.
1. The license is strongly bound to a unique device ID to prevent copying to other devices.
2. Supports setting an expiration date; licenses must be re-authorized upon expiration.
3. Supports different authorization verification modes, including online and offline verification.

## Device Information Acquisition and Operation
1. Obtain the unique hardware ID of the Miaosuan 3.  
"cat /factory/device_id.txt" or "cat /sys/firmware/devicetree/base/serial-number"
2. Time Acquisition
The aircraft's GPS UTC time can be obtained through the PSDK data subscription.
1. User Interface Input
The Pilot control provided by the PSDK allows users to enter the activation code in text on the remote controller Pilot or Sikong 2 interface.

## Storage and Security Recommendations
1. Store the license file in the application's private directory to prevent user copying and replacement. Reference paths include "open_app/xxx application/xxx custom directory" or "/home/dji/xxx custom directory."
2. Encrypt or obfuscate the file to prevent direct reading or modification.
3. Obfuscate the core algorithm logic to prevent reverse engineering (e.g., LLVM obfuscation).

## Solution Selection Recommendations
1. Online Mode (Asymmetric Algorithm):  
Advantages: Highly secure, private keys are not on the device.  
Disadvantages: Network-dependent, unavailable when the user is offline.

2. Offline Mode (Symmetric/Private Algorithm):  
Advantages: Suitable for offline use, local activation and verification are possible.  
Disadvantages: Security depends on the strength of the algorithm, requiring protection against reverse engineering.
