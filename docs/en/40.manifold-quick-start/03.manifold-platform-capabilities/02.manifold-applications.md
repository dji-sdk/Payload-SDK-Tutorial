

## What Is a Manifold Application (DPK)?

A Manifold application is an application developed for the Manifold 3 platform based on the DJI PSDK interface. It is referred to as a **Manifold Application**, and its standard distribution format is a **DPK application**.

Developers can cross-compile applications developed with PSDK and use the application packaging tool `build_dpk.sh` provided by DJI to generate an installation package with the `.dpk` extension (that is, a Manifold application installation package). End users can then use the DJI Pilot App or the built-in application management tool `dji_app_ctl` on Manifold 3 to install, update, and uninstall Manifold applications.

With the DPK application format, Manifold 3 standardizes and encapsulates the application deployment and management process. Compared with Manifold 2, users no longer need to be familiar with Linux commands, which significantly lowers the barrier to on-site deployment and use, and improves the usability and deliverability of industry applications.

## Relationship Between Manifold and PSDK

PSDK serves as the interface layer between Manifold applications and DJI flight platforms, exposing the capabilities of the flight platform. Manifold 3 provides the computing power, system environment, and runtime carrier for PSDK applications. Developers build applications based on PSDK and deploy and run them on Manifold 3 to enable interaction and control with the flight platform.

## Manifold Application Development Specifications and Guidelines

Developing Manifold applications requires the use of DJI PSDK and compliance with the application development and deployment specifications of the Manifold 3 platform. To ensure that an application can be packaged, installed, and run successfully, developers must follow the development constraints below and are advised to follow the recommended practices.

### Development Constraints

* **Dependency Library Restrictions**  
  Manifold applications support only **static libraries** and do not support dynamic libraries.  
  The use of dynamic libraries may cause application packaging, installation, or startup failures.

* **Configuration File Path Requirements**  
  Relative paths must be used for reading and writing configuration files in the application source code. Absolute paths are not allowed.  
  Otherwise, the application may fail to install or start properly.

* **`app.json` Configuration File Requirements**  
  Developers must create the application-level configuration file `app.json` in accordance with the specification.  
  You can refer to the sample file in the DJI PSDK software development kit:  
  `samples/sample_c/platform/linux/manifold3/app_json/app.json`  
  For the detailed `app.json` specification, refer to:  
  `samples/sample_c/platform/linux/manifold3/app_json/README.md` in the PSDK software development kit, or the section <a href="../build-dpk.html#app-json-configuration-file" target="_blank" rel="noopener noreferrer">app.json Application-Level Configuration File Description</a>.

### Development Recommendations

* **Log File Directory Convention**  
  It is recommended that application log files be written to the `data/logs` directory.  
  Otherwise, functions such as application data clearing and log export may not work properly.

* **Media File Directory Convention**  
  It is recommended that media files generated or used by the application be stored in the `data/media` directory.  
  Otherwise, application data clearing and other media-related functions may be affected.
