## Application Installation and Update
Supports installing/updating Manifold third-party applications to Manifold 3 devices through PILOT and FlightHub 2.


### Installing/Updating Third-party Applications Using PILOT
The operation of installing/updating third-party applications through PILOT is as follows. After obtaining the DPK application installation package, copy the installation package file (i.e. xxx.dpk file) to the remote control through a USB cable or SD card, open the PILOT Manifold 3 interface, and click **Application Management** to enter the Manifold 3 application management page.

### Installing/Updating Third-party Applications Using FlightHub 2
The process for installing/updating third-party apps via FLIGHTHUB 2 involves the following steps:

1. Log in to [FLIGHTHUB 2](https://es-flight.dji.com/login) using a web browser.
2. Access the Devices interface.
3. Search for and select the desired device.
4. Choose the "Device Maintenance" option for that device.
5. Navigate to the "Manifold 3 Application Management" within the Device Maintenance.
6. In the application management interface, click "+ Import" (located in the upper right corner).
7. Select the xxx.dpk file to install or update the DPK application.

## Application Start and Stop
Once the DPK application has been installed or updated, you can start and stop the application, as well as view its current running status. You can start, stop, and view the running status of the application on both PILOT and FlightHub 2.

### Starting/Stopping Applications Using PILOT

Enter the PILOT APP and click **Enter Camera View**.
<div>
<div align=left>
<img src="https://terra-1-g.djicdn.com/71a7d383e71a4fb8887a310eb746b47f/psdk/app_mgr/%40%401752231188575.png" width=1000 style="height:auto;"/>
</div>
</div>

On the left side of the interface, previously installed/updated applications in the Manifold 3 device will appear. Click any application icon to start the application. If the application starts successfully, the corresponding application icon will turn yellow, and the application runtime scene will also appear in the ﬂight interface. Based on this, you can test application functions. Click the application icon again to close the application, and the corresponding icon will turn gray, indicating the application is no longer running.

## Application Uninstallation

If you no longer want to use an installed/updated third-party DPK application and want to delete it from the Manifold 3 device, the operation is: enter the Manifold 3 application management interface > select the application you want to delete > click the application uninstall option to delete the corresponding application.

## Application Data Clear
You can clear these data generated during third-party application operation such as Log ﬁles and various data by selecting the data clear option in the Manifold 3 application management interface.

## Application Log Export
After entering the Manifold 3 application management interface, you can export third- party application logs to facilitate developers' after-sales problem analysis.
- Export logs on PILOT: Insert a U disk or card reader into the general USB port of Manifold, then click the "Log Export" option on PILOT to export application logs from Manifold to the U disk.
- Export logs on FlightHub 2: Select the application you want to export logs from, click the "Log Export" option to export application logs locally.
- U disk or SD card supported formats: exFAT, NTFS, FAT32.

## Setting Auto-start on Boot

- Supports modifying application auto-start on boot function on PILOT. Operation on PILOT: click Manifold 3 > Application Management > More for each application > Set as Auto-start on Boot option to set the current application to auto-start on boot.  
- Applications already set to auto-start support canceling auto-start on boot, with the same operation method as above.  
- Only one application can be set to auto-start on boot, and the later set application has the auto-start on boot priority.  

:::tip Note: 
The above method only supports setting auto-start for DPK applications. If you do not need to distribute Manifold 3 DPK applications as software, or if your application depends on complex environments like ROS / is not convenient to package as DPK, you can refer to <a href="../../faq.html#how-to-configure-auto-startup-using-systemd-user" target="_blank" rel="noopener noreferrer">Setting Auto-start on Boot Using systemd --user Method</a> in the Appendix.
:::
